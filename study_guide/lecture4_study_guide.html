Lecture 3 建立了损失函数的概念——SVM loss 和 Softmax loss 告诉我们"当前参数有多差"。但 loss 函数本身只是一把尺子，它不会自动帮你找到最优参数。Lecture 4 回答的就是"如何找到让 loss 最小的参数 W"这个核心问题。

PART 02

梯度下降：损失地形上的"下山问题"

对应 Slides 6–15

把损失函数 L(W) 想象成一片高维地形——每一组参数 W 对应地形上的一个点，这个点的"海拔"就是 loss 值。训练的目标就是找到"最低谷"。

在二维参数空间中，梯度 ∇L 指向 loss 上升最快的方向；反向 -∇L 就是下降最快的方向。
梯度下降的通用公式
Wt+1 = Wt − α · ∇L(Wt)
α 是学习率（步长），∇L 是梯度（方向）

三种梯度下降变体（Slide 12）

名称用多少数据算梯度优缺点
Batch GD 全部训练数据 梯度精确但极慢；N=100 万时每步都不可行
SGD 单个样本 每步极快但梯度噪声大，震荡严重
Mini-batch SGD 32–256 个样本 折中方案，GPU 友好，实际中的"SGD"默认指这个
为什么 mini-batch 是黄金标准 单样本 SGD 的梯度噪声太大，训练很不稳定；全批量 GD 每步太慢，跟不上数据规模的增长。Mini-batch 同时解决了两个问题：噪声可控（多个样本平均会抵消随机性）+ GPU 并行（矩阵运算天然适合 batch 处理）。你在 A1/A2 中用的 batch_size=200 就是这个道理。
PART 03

Vanilla SGD 与它的四个致命问题

对应 Slides 16–25

Vanilla SGD 就是最朴素的梯度下降：

# 伪代码
while True:
    dx = compute_gradient(x)     # 算当前点的梯度
    x = x - learning_rate * dx   # 沿反方向走一小步

简洁优美——但在实际训练中，SGD 暴露出四个严重问题。Justin 在 slide 中对每个问题都给出了直观的几何解释。

问题一：狭长山谷中的震荡（Slides 17–18）

想象一个"浅宽的峡谷"——沿一个方向坡度陡峭，沿另一个方向坡度平缓。这在深度学习中极其常见，因为不同参数的敏感度往往相差悬殊。

典型的病态地形：SGD 在陡峭方向过度反应，在平缓方向进展太慢
根本原因 所有参数共享同一个学习率。陡峭方向需要小学习率（否则会越过最低点反复震荡），平缓方向需要大学习率（否则爬得太慢）——两个矛盾的要求，SGD 无法同时满足。

问题二：局部最小值与鞍点（Slides 19–22）

SGD 只看当前点的梯度，一旦梯度 = 0 就停下。问题是在高维空间中，梯度为零的点大多是鞍点（saddle point）而非真正的最小值——在某些方向是上坡，在另一些方向是下坡。

❌ 局部最小值陷阱

在低维（2D, 3D）空间中，局部最小值确实是个问题。但在高维（百万参数）空间中，"所有方向都是上坡"的点极其罕见。

⚠️ 鞍点才是真敌人

高维空间中鞍点无处不在（Dauphin et al. 2014）。SGD 在鞍点附近梯度极小，训练会"假装收敛"然后卡住。

问题三：梯度噪声（Slide 23）

Mini-batch 的梯度是对真实梯度的噪声估计。在正常区域这不是问题，但在接近最优点的平坦区域，噪声会让优化轨迹"醉酒般"地乱走，延迟收敛。

问题四：一视同仁的学习率（隐含问题）

假设 W 中有些参数更新很频繁（对应常见特征），有些更新很稀疏（对应罕见特征）。SGD 对两者用同一个学习率——常见特征的权重可能很快收敛，罕见特征的权重却几乎没动。这种差异无法用全局学习率处理。

四个问题指向两种修复思路 问题 1、2、3 → 需要时间上的平滑（让更新方向不要过度依赖当前一步的梯度）→ Momentum。
问题 1、4 → 需要空间上的差异化（让不同参数有不同的有效学习率）→ AdaGrad / RMSProp。
Adam 同时采用了这两种思路。
PART 04

修复一：Momentum 引入"惯性"

对应 Slides 26–32

Momentum 的灵感来自物理学——想象一个有质量的小球在地形上滚动。它不会瞬间改变方向，而是会沿着历史运动方向保持惯性。

SGD + Momentum
vt+1 = ρ · vt + ∇L(Wt)
Wt+1 = Wt − α · vt+1
v 是"速度"（过去梯度的累积），ρ 通常取 0.9（90% 惯性）
Momentum 让"速度"积累，抵消了震荡分量（左右来回的方向互相抵消），放大了一致分量（向最优点的方向不断累积）

为什么 Momentum 有效（Slide 30 的三个视角）

①
抵消震荡 在"左震右震"的方向上，相邻时刻的梯度方向相反，累积后互相抵消 → 震荡分量被平滑掉。
②
加速穿越平坦区 在平缓方向上，虽然每步梯度都很小，但它们方向一致，累积后速度越来越大 → 平坦方向被加速。
③
逃离鞍点 在鞍点附近梯度接近零，但 Momentum 保留了到达鞍点之前的速度，可以靠惯性继续前进 → 不会"卡死"。
ρ = 0.9 的物理直觉 ρ 决定了"记住多少历史"。ρ = 0.9 意味着"90% 保留历史速度 + 10% 加入当前梯度"——历史在几十步后才会被完全冲淡。对应指数衰减的"半衰期"大约是 7 步。在实践中，ρ 取 0.9 到 0.99 几乎涵盖所有场景。
PART 05

进阶：Nesterov Momentum

对应 Slides 31–33

Nesterov Momentum 是对标准 Momentum 的一个小改良，在凸优化理论上有更好的收敛速度保证。核心变化是"先走一步再算梯度"。

标准 Momentum

在当前位置 Wt 算梯度
↓
把"速度 + 梯度"加起来走

可能由于惯性"过冲"——跑过最优点后才发现梯度变了

Nesterov Momentum

先按惯性 ρ·v 走到"预测位置"
↓
在预测位置算梯度，再修正

具有"前瞻性"——如果发现前方要上坡，提前修正

实践建议 Nesterov 在理论上比标准 Momentum 好，但实际中差别不大。对于大多数作业和项目，用标准 SGD + Momentum 已经足够。Nesterov 主要在一些对收敛速度敏感的场景（如 GAN 训练）中被使用。
PART 06

修复二：AdaGrad 引入"自适应"

对应 Slides 34–37

Momentum 解决了"时间上的平滑"问题，但没有解决"不同参数需要不同学习率"的问题。AdaGrad 从完全不同的角度切入：为每个参数追踪它自己的梯度历史。

AdaGrad 自适应

gt = ∇L(Wt)   当前梯度

vt = vt-1 + gt2   所有历史梯度平方的累积

Wt+1 = Wt − &frac;α / (√vt + ε) · gt

关键：gt2 是逐元素平方；vt 和 W 形状相同，每个参数有自己的"历史大小"

核心洞见：自适应学习率（Slide 35）

分母 √vt 是该参数历史梯度大小的度量：

📈
梯度一直很大

vt 增长快
√vt 很大
有效学习率自动变小

📉
梯度一直很小

vt 增长慢
√vt 很小
有效学习率自动变大

回到 Part 3 的狭长山谷问题——陡峭方向（梯度大）的学习率会被自动压低，平缓方向（梯度小）的学习率会被自动放大。这正是我们想要的！

AdaGrad 解决了什么 SGD 的问题一（狭长山谷震荡）和问题四（参数间差异化需求）——两个问题都靠"逐参数自适应学习率"同时解决。每个参数的 vt 独立追踪，互不干扰。
PART 07

AdaGrad 的缺陷与 RMSProp 的修复

对应 Slides 37–39

AdaGrad 听起来完美，但有一个致命缺陷——vt 只增不减。

🐛 AdaGrad 的问题

vt 是从训练开始到现在所有梯度平方的累积和。训练越久，vt 越大，有效学习率 α/√vt 越小——最终趋近于零，模型在到达最优解前就"冻住"了。

➜
🔧 RMSProp 的修复

把"累积和"改成"指数滑动平均"——远古的梯度信息自动衰减，vt 不会无限膨胀。解决了学习率坍缩问题。

RMSProp 修复版

vt = β2 · vt-1 + (1 − β2) · gt2   滑动平均代替累积

Wt+1 = Wt − α / (√vt + ε) · gt

β2 通常取 0.999，意味着"99.9% 保留历史 + 0.1% 加入当前"

为什么 RMSProp 没有论文 RMSProp 从未正式发表！它是 Geoffrey Hinton 在 2012 年一节 Coursera 课程里提出的，引用格式就是"Hinton's Coursera Lecture 6"。尽管如此，它是深度学习历史上最有影响力的算法之一——因为它为后来的 Adam 铺平了道路。
PART 08

Adam：两种思想的合体

对应 Slides 40–44

到目前为止我们有了两种互补的改良：

Momentum（一阶矩）

时间上平滑梯度的方向，给优化"惯性"

RMSProp（二阶矩）

空间上给每个参数不同的步长

一个自然的问题是——能不能同时用上这两种思想？ Adam（Adaptive Moment Estimation）给出了答案。

Adam 合体版

mt = β1 mt-1 + (1 − β1) gt   一阶矩：梯度滑动平均（Momentum）

vt = β2 vt-1 + (1 − β2) gt2   二阶矩：梯度平方滑动平均（RMSProp）

m̂t = mt / (1 − β1t),   v̂t = vt / (1 − β2t)  偏差校正

Wt+1 = Wt − α · m̂t / (√v̂t + ε)

把更新公式拆开看

Adam 的更新 = 全局步长 × Momentum 方向 / RMSProp 步幅缩放
理解 Adam 的关键 Momentum 关心的是"哪个方向"（时间维度的平滑）。RMSProp 关心的是"每个参数多大步"（空间维度的缩放）。这两件事是正交的——你可以独立使用其中一个，也可以同时使用两个。Adam 就是同时使用两个的版本。
PART 09

偏差校正：Adam 最精妙的细节

对应 Slide 43

Adam 公式中的 m̂t = mt / (1 - β1t) 这一步叫偏差校正。很多人第一次看以为是技术细节可以忽略，但它实际上是 Adam 论文最精妙的贡献。

问题：零初始化偏差

训练刚开始时，m0 = v0 = 0。看看第一步会发生什么：

第一步（t = 1）的 m1
m1 = 0.9 · 0 + (1 − 0.9) · g1 = 0.1 · g1
m1 只有真实梯度的 10%！

对于 vt 更严重：v1 = 0.001 · g12——只有真实值的 0.1%。这会导致什么？

没有偏差校正时的怪异行为

Adam 的更新 α · m / √v。训练初期，m 被低估 10 倍，v 被低估 1000 倍——m / √v = 10g / √(1000 g²) ≈ 0.32 g / |g|，步长反而会被异常放大或方向不稳，训练初期表现很差。

偏差校正的工作原理

我们知道 mt 被低估了。具体低估多少？数学上可以证明：

偏差的精确表达
E[mt] = (1 − β1t) · E[真实梯度]
mt 的期望值比真实值少了因子 (1 - β1t)

既然知道少了多少，直接除回去就能修正：

校正在早期放大估计值（补偿零初始化的偏差），后期自动消失（因为 (1-β1t) → 1）
这就是 Adam 的魔力之一 偏差校正让 Adam 的训练从第一步就"step size 稳定"——不需要 warmup 期，不需要手动调整早期学习率。这是 Adam 几乎可以开箱即用（default lr = 3e-4 即可）的关键原因之一。你在 A2 中如果手写了 Adam，必须记得加上这个校正。
PART 10

二阶方法：牛顿法与 L-BFGS

对应 Slides 45–52

到目前为止的所有方法（SGD, Momentum, AdaGrad, RMSProp, Adam）都只用了一阶信息——梯度。但导数还有更高阶的信息可以利用。

牛顿法：理想的二阶方法

牛顿法的更新（Slide 47）
Wt+1 = Wt − H−1 · gt
H 是 Hessian 矩阵（二阶偏导矩阵），告诉我们地形的"曲率"

没有学习率！牛顿法根据曲率自动计算最优步长。在二次函数上，牛顿法一步就能到最优解。

为什么不用牛顿法 对于有 D 个参数的模型，Hessian 是 D×D 的矩阵。CIFAR-10 线性分类器就有 D ≈ 30,000 参数，Hessian 是 30000×30000 ≈ 109 个元素——存不下。现代神经网络动辄上亿参数，Hessian 是天文数字。而且还要算它的逆 — 完全不可行。

L-BFGS：用一阶信息近似二阶效果

L-BFGS（Limited-memory Broyden–Fletcher–Goldfarb–Shanno）的核心思想：不显式存储 Hessian，而是用最近几步的梯度变化来近似它。

①
只保留最近 m 步 通常 m = 5~20，内存开销从 O(D²) 降到 O(mD)——可以接受了。
②
隐式构造 H-1 利用这些梯度历史，通过特殊算法近似 Hessian 的逆，而不是显式存储。
L-BFGS 的限制 L-BFGS 要求 loss 和梯度是精确的、确定性的。但深度学习用 mini-batch，梯度本身带噪声。L-BFGS 依赖前后梯度的精确差异来估计曲率，噪声会让估计崩溃。所以 Justin 在 slide 上说："If you can afford full batch updates, try L-BFGS. Don't forget to disable all sources of noise."

L-BFGS 的实际应用

L-BFGS 在深度学习中极少使用，但在某些特定场景中非常有用：

  • 风格迁移（Style Transfer）：图像固定、参数量不大、可以 full batch — A6 会遇到
  • 小规模凸优化问题：逻辑回归等场景，收敛极快
  • 超参数优化：有些自动调参工具内部用 L-BFGS

优化器选择实战指南

对应 Slides 53–55

优化器演进时间线

1950s · Robbins & Monro
SGD

最基础的随机梯度下降

1964 · Polyak
Momentum

引入"惯性"，解决震荡问题

2011 · Duchi et al.
AdaGrad

首次引入逐参数自适应学习率

2012 · Hinton (Coursera)
RMSProp

用滑动平均修复 AdaGrad 的学习率坍缩

2014 · Kingma & Ba
Adam

Momentum + RMSProp + 偏差校正的完整组合

2017–2024
AdamW, LAMB, Lion, ...

针对不同场景的 Adam 变体，但 Adam 本身仍是默认选择

Justin 的选择建议（Slide 54）

场景推荐为什么
大多数情况 Adam with lr = 3e-4 开箱即用，对超参数不敏感
图像分类（ResNet 等） SGD + Momentum 精调后往往比 Adam 泛化性能更好
Transformer / NLP Adam + Warmup Transformer 训练对 Adam 特别依赖
能够 full batch 更新 L-BFGS 收敛极快，但需要关闭所有噪声源
GAN / 对抗训练 Adam with β1=0.5 特殊场景需要特殊调整
"3e-4" 的由来 Andrej Karpathy 有一条著名推文："3e-4 is the best learning rate for Adam, hands down."（后来他澄清是半开玩笑）。但实际上，3e-4 确实是 Adam 在大多数场景下一个非常合理的起点。如果没有时间调参，用 Adam + lr=3e-4 几乎不会出大错。
Adam vs SGD+Momentum 的"泛化之争" 近年来研究发现，在图像分类任务中，精心调节的 SGD+Momentum 能得到比 Adam 更好的测试精度（而不仅是训练精度）。Wilson et al. (2017) 的论文 "The Marginal Value of Adaptive Gradient Methods" 指出，自适应方法容易收敛到"sharp minima"，泛化能力较差；SGD 倾向于"flat minima"，泛化更好。这就是为什么 ResNet 等经典 CNN 仍然用 SGD 训练的原因。但对于 Transformer，Adam 几乎是唯一选择。

Vibe Coding 时代的竞争力

场景：AI 生成的代码用了默认 Adam，但训练不收敛 AI 写的代码八成是 torch.optim.Adam(model.parameters(), lr=1e-3)。但如果 loss 不下降，AI 不会知道该改什么。你需要判断：是学习率问题？梯度爆炸？初始化问题？这种诊断能力来自你理解每个优化器在做什么，而不是"Adam 比 SGD 好"这种口号。
场景：模型在训练集上表现好，测试集上差 AI 会建议"加更多数据 / 加 dropout"。但知道了 Adam 倾向 sharp minima 的特性后，你可以尝试把优化器换成 SGD+Momentum——这是很多论文里提升图像分类泛化的关键技巧。这种"跳出工具链思维"的能力是 AI 无法替代的。
场景：面试中被问"Adam 为什么需要偏差校正" 这是机器学习工程师面试的经典问题。浅层回答是"防止零初始化偏差"；深层回答是能够推导出 E[mt] = (1-β1t) · E[g]、解释为什么早期步长会异常、演示用 Python 手写 Adam。这种数学 + 代码的双重理解让你在面试中脱颖而出。
