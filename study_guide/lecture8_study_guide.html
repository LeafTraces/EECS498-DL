PART 01

ImageNet 竞赛与 CNN 发展史

对应 Slides 25–27, 55, 65–66, 75, 108

要理解 CNN 架构为什么是今天这个样子，你必须先理解推动它们演化的竞赛：ImageNet Large Scale Visual Recognition Challenge (ILSVRC)。这个竞赛要求模型在 1000 个类别、约 120 万张训练图像上进行图像分类，评价指标是 top-5 错误率（即正确标签是否在模型预测概率最高的 5 个类别中）。

Justin 在 Lecture 8 中用了一张贯穿始终的柱状图（Slides 25, 55, 65, 66, 75, 108），每讲完一个架构就在图上添加一根新的柱子。这张图本身就是整个深度学习发展史的缩影：

2010–2011
浅层方法（Shallow）
Top-5 error: 28.2% → 25.8%。手工特征 + SVM，没有深度学习。
2012
AlexNet — 8 层
Top-5 error: 16.4%。第一个深度学习方法，错误率暴降约 10 个百分点。深度学习时代的开端。
2013
ZFNet — 8 层
Top-5 error: 11.7%。本质上是调参后的 AlexNet（改了滤波器大小和步幅），Justin 在课上跳过了它。
2014
VGGNet — 19 层  |  GoogLeNet — 22 层
Top-5 error: 7.3%（VGG）/ 6.7%（GoogLeNet）。首次突破 10%，"深度"开始成为关键词。
2015
ResNet — 152 层
Top-5 error: 3.6%，首次超越人类水平（~5.1%）。残差连接使超深网络成为可能。
宏观趋势 从 2012 到 2015，仅仅三年时间，网络层数从 8 增长到 152，错误率从 16.4% 降到 3.6%。每一次突破都伴随着一个关键的架构创新——而不仅仅是"把网络做大"。理解为什么这些创新有效，比记住架构本身更重要。

逐层计算训练（最重要的技能！）

Justin 在 Slides 36–46 花了整整 10 张 slides 一步步填充这个表格。让我以 conv1 为例，教你这个计算方法（Slide 36 的练习题）：

conv1 完整计算演示

已知：Input 3×227×227，64 个 11×11 filters，stride=4，pad=2

输出尺寸：H' = (227 − 11 + 2×2) / 4 + 1 = 220/4 + 1 = 56
→ Output: 64×56×56

Memory（存储激活值）：64 × 56 × 56 × 4 bytes = 802,816 bytes ≈ 784 KB
（每个浮点数 4 bytes，需要存储激活值用于反向传播）

Parameters：64 × 3 × 11 × 11 + 64(bias) = 23,296 ≈ 23K

FLOPs：64 × 3 × 11 × 11 × 56 × 56 = 72,855,552 ≈ 73M
（每个输出位置做一次 Cin×K×K 的点积，共 Cout×H'×W' 个位置）

练习建议 在看 Slide 36 之后的每一张 slide 之前，先自己算出那一层的数值，然后翻页验证。当你能对着一个任意的 conv/pool/fc 层秒算 output size、memory、params、FLOPs 时，这个技能就内化了。这在面试中和论文阅读中都极其有用。

三个关键发现（Slide 50）

Slide 50 是 AlexNet 部分的灵魂——Justin 画了三张柱状图，揭示了一个极其重要的分布规律：

Memory 分布

绝大部分内存消耗在前几层卷积（conv1 占 784KB，而 fc8 只占 4KB）。原因：前面的层空间分辨率大（56×56），虽然通道数少，但 H×W 很大。

Params 分布

绝大部分参数在全连接层（fc6 占 37.7M，而所有 conv 层加起来才 2.5M）。原因：fc6 的输入是 9216 维、输出是 4096 维，参数量 = 9216×4096 ≈ 37.7M。

FLOPs 分布

绝大部分计算量在卷积层（conv2 占 224M FLOPs，fc6 只占 38M）。原因：卷积的 FLOPs = Cout×Cin×K²×H'×W'，空间尺寸 H'×W' 是乘数。

这个分布规律至关重要 Memory：前面（大 H×W）  |  Params：FC 层  |  FLOPs：卷积层

这个规律不仅适用于 AlexNet，它是所有 CNN 的普遍规律。后续架构的每一次创新，都可以理解为在这三个维度上的优化：VGG 牺牲了 params 和 FLOPs 换取深度；GoogLeNet 通过 1×1 卷积和 GAP 同时压缩了三者；ResNet 用残差连接解锁了深度的上限。

AlexNet 的历史创新

虽然从今天的视角看，AlexNet 的设计有些"原始"（11×11 大卷积核、不规则的通道数），但它在 2012 年引入了多项开创性技术：

ReLU 激活函数——首次在大规模网络中使用，替代了 Sigmoid/Tanh，训练速度快 6 倍。Dropout 正则化——在 fc6 和 fc7 中使用 p=0.5 的 Dropout，有效防止过拟合。数据增强——随机裁剪、水平翻转、颜色抖动，扩大了有效训练集。GPU 训练——在两块 GTX 580（各 3GB 显存）上并行训练，Slide 28 的原始论文插图清楚展示了两个 GPU 的分工。

为什么 AlexNet 用 11×11 大卷积核？ 这不是因为 11×11 是最优选择，而是受限于当时的计算能力。第一层用大 kernel + 大 stride (4) 的目的是快速下采样，把 227×227 降到 56×56，否则后续层的计算量会爆炸。后来 VGG 证明了用多层 3×3 小卷积替代大卷积核是更好的设计，但这需要更多的计算力——2012 年的 GPU 做不到。
PART 03

VGGNet (2014)：简洁之美

对应 Slides 57–65

VGGNet 由 Oxford 的 Simonyan 和 Zisserman 提出。如果说 AlexNet 是一个充满手工调参痕迹的"草创之作"，VGG 则是第一个遵循系统性设计原则的架构。

三条黄金设计规则（Slide 57, 62）

VGG Design Rules

规则 1：所有卷积层都使用 3×3, stride 1, pad 1（保持空间尺寸不变）。

规则 2：所有 max pooling 都使用 2×2, stride 2（空间尺寸减半）。

规则 3：每次 pool 后，通道数翻倍（64 → 128 → 256 → 512）。

这三条规则定义了至今仍被广泛使用的 CNN 设计范式。Slide 62 进一步量化了 pool 前后的变化：

Pool 之前

Input: C × 2H × 2W
一层 3×3 Conv(C→C) 的：
Memory: 4HWC
Params: 9C²
FLOPs: 36HWC²

Pool 之后（C 翻倍）

Input: 2C × H × W
一层 3×3 Conv(2C→2C) 的：
Memory: 2HWC（减半）
Params: 36C²（4倍）
FLOPs: 36HWC²（不变！）

为什么通道数翻倍的设计很聪明？ pool 让 H、W 各减半，面积减为 1/4。如果通道数不变，每层的计算量会急剧下降，网络后半段几乎不做任何有意义的计算。通道翻倍（C→2C）使 FLOPs 大致恒定，让每一层都承担相近的计算负担——这是一种朴素但有效的"负载均衡"。

VGG-16 / VGG-19 的 5 stage 结构

Slide 57 给出了完整的层级对比（从底部 Input 向上读）：

VGG 的贡献与局限

VGG 的核心贡献 证明了"深度"的价值——在相同的设计框架下，从 16 层加到 19 层能持续提升性能（7.3% → 略低）。

确立了 3×3 卷积 + 5 stage 的标准范式——后来的 ResNet 直接继承了这个 5 stage 框架。

VGG 特征的强迁移性——VGG 的预训练特征在目标检测、分割等下游任务上表现极好，是很长一段时间内的"默认 backbone"。
VGG 的严重问题 138M 参数，其中绝大部分在三个 FC 层中（FC 4096 占了约 120M）。13.6 GFLOP，计算量是 AlexNet 的 19.4 倍。这使得 VGG 在部署和训练时都非常昂贵。后来的 GoogLeNet 和 ResNet 都在试图解决这个效率问题。
PART 04

GoogLeNet / Inception (2014)：效率至上

对应 Slides 66–75

GoogLeNet（也叫 Inception v1）由 Google 的 Szegedy 等人提出。它的设计哲学与 VGG 截然不同——不追求简洁，追求效率。Justin 在 Slide 67 精炼地总结为："Many innovations for efficiency: reduce parameter count, memory usage, and computation."

核心创新 1：Inception Module

Inception Module 的设计动机是：我们不知道每一层应该用多大的卷积核（3×3？5×5？还是只做池化？），那就全部都用，让网络自己学习最优的组合。

在一个 Inception Module 内部，同一个输入被并行送入四条路径，输出在通道维度上拼接（concatenate）：

1×1 conv
3×3 conv
1×1 conv
5×5 conv
1×1 conv
1×1 conv
3×3 pool
↑ ↑ ↑ ↑
Previous layer


四条路径的输出在 channel 维度上 concatenate → 送入下一个 Inception Module
Inception Module with 1×1 bottleneck（关键：每条有 3×3 或 5×5 的路径前都加一个 1×1 卷积降维）
1×1 卷积的 "瓶颈" 作用 如果不加 1×1 瓶颈，直接对高通道数的输入做 5×5 卷积，计算量会爆炸。以输入 192×28×28 为例：

无瓶颈：直接用 32 个 5×5 conv → FLOPs = 192×5×5×32×28×28 ≈ 120M
有瓶颈：先用 16 个 1×1 conv 降到 16 通道，再用 32 个 5×5 conv → FLOPs ≈ 12.4M

减少了约 10 倍 计算量！1×1 卷积充当了"降维通道"——先把 192 维特征压缩到 16 维，再做空间卷积。

核心创新 2：Aggressive Stem（Slide 68, 70）

GoogLeNet 在网络开头使用了"激进的 stem 网络"来快速下采样：从 224×224 经过 conv(7×7, stride 2) + pool + conv(1×1) + conv(3×3) + pool，迅速降到 192×28×28。

Slide 70 的对比数据触目惊心——仅 stem 部分（从 224 到 28 的分辨率）：

GoogLeNet StemVGG-16 同等范围
Memory7.5 MB42.9 MB（5.7×）
Params124K1.1M（8.9×）
MFLOP4187485（17.8×）

核心创新 3：Global Average Pooling（Slide 73）

VGG 的致命弱点是最后的 FC 层吞掉了大部分参数（120M+）。GoogLeNet 用了一个极其简洁的替代方案：

Global Average Pooling (GAP) 消灭 FC 层 在最后一个 Inception Module 输出 1024×7×7 之后，用一个 7×7 的 Average Pooling 把空间维度压到 1×1，得到 1024×1×1，然后只需一个 1024→1000 的 FC 层就能输出类别分数。

对比 VGG：9216→4096→4096→1000 的三层 FC 贡献了 ~120M 参数。
GoogLeNet：1024→1000 的一层 FC 只有 ~1M 参数。

总参数量只有 5M（VGG 的 1/28！），而 top-5 错误率更低（6.7% vs 7.3%）。
GAP 的现代影响 Global Average Pooling 后来成为几乎所有现代 CNN 的标配（ResNet、EfficientNet、ConvNeXt 都用它）。它的思想是：让网络的最后几层专注于"学习好的特征"，而不是用巨大的 FC 层做"特征到类别的映射"。在 Transformer 中，[CLS] token 的作用与 GAP 类似——从序列中聚合一个全局表示用于分类。
PART 05

ResNet (2015)：最重要的架构创新

对应 Slides 75–100

ResNet 由微软亚洲研究院的何恺明等人提出，是深度学习史上影响力最大的单篇论文之一。Justin 花了 20+ slides 讲它，因为它引入的残差连接至今仍是几乎所有深度网络的基础。

问题：深网络反而更差？（Slide 80）

有了 BatchNorm，我们终于能训练 10+ 层的网络了。但当层数继续增加时，出现了一个违反直觉的现象：

退化问题（Degradation Problem） 56 层的网络比 20 层的网络训练误差更高（不是测试误差！），这说明这不是过拟合，而是欠拟合——更深的网络连训练数据都学不好。

这很奇怪，因为理论上 56 层网络至少可以做到和 20 层一样好——让多余的 36 层学成恒等映射（identity function）就行了。但实际上，网络很难学到恒等映射——把一堆非线性变换的参数调到恰好实现"什么都不做"，是一个很难的优化问题。

解决方案：残差连接（Slide 83）

Slide 83 是本讲最重要的一张图：Plain Block vs Residual Block。

"Plain" Block

x ↓ conv ↓ relu ↓ conv ↓ H(x)

网络需要直接学习期望的映射 H(x)。如果 H(x) ≈ x（恒等映射），就需要让 conv 的权重恰好满足这个条件——很难。

Residual Block

x ↓          ↘ (shortcut) conv ↓ relu ↓ conv ↓          ↗ (add) F(x) + x → relu

网络只需学习残差 F(x) = H(x) − x。如果最优映射接近恒等，只需把 F(x) 推向 0——把所有权重推向零比学恒等映射容易得多！

Output = F(x) + x

反向传播时：∂L/∂x = ∂L/∂Output × (∂F(x)/∂x + I)
那个 I（单位矩阵）保证梯度永远不会完全消失

Basic Block vs Bottleneck Block（Slide 93）

Slide 93 展示了 ResNet 的两种 block 设计：

"Basic" Block (ResNet-18/34)

Conv(3×3, C→C) ↓ Conv(3×3, C→C)

FLOPs = 2 × 9HWC² = 18HWC²

"Bottleneck" Block (ResNet-50/101/152)

Conv(1×1, 4C→C) ← 降维 ↓ Conv(3×3, C→C) ↓ Conv(1×1, C→4C) ← 升维

FLOPs = 4C²+9C²+4C² = 17HWC²
FLOPs 几乎相同，但 block 内有 3 层而非 2 层 → 可以在相同计算量下堆更多层！

Bottleneck 的精髓 1×1 conv 先把 4C 通道压缩到 C，在低维空间做计算昂贵的 3×3 卷积，再用 1×1 conv 升回 4C。这与 GoogLeNet 的 Inception Module 使用 1×1 瓶颈是同一个思想——在低维做重活，在高维传递信息。

ResNet 的历史地位（Slide 98）

ResNet 横扫了 2015 年 ILSVRC 和 COCO 竞赛的所有五个赛道的第一名：ImageNet Classification、ImageNet Detection、ImageNet Localization、COCO Detection、COCO Segmentation。Slide 98 引用了 Yann LeCun 对 152 层 ResNet 的评价——"Ultra-deep"。

改进：Pre-Activation ResNet Block（Slide 100）

He et al. 在 2016 年的后续论文中提出了一种改进的 block 结构——把 BN 和 ReLU 移到 Conv 前面：

Original: Conv → BN → ReLU

shortcut 路径上最后经过一个 ReLU，信息有损。

Pre-Activation: BN → ReLU → Conv

shortcut 是纯粹的恒等映射，梯度可以完全无损地流过。

Slide 100 显示这带来了少量精度提升（ResNet-152: 21.3 → 21.1, ResNet-200: 21.8 → 20.7），但 Justin 也诚实地指出 "not actually used that much in practice"——工程实践中多数人仍用原始版本。

PART 07

设计规律与现代视角

贯穿整个 Lecture

四个普适设计原则

从 AlexNet 到 ResNet 的演进中，有四条经验不断被验证并沿用至今：

原则 1：空间递减 + 通道递增。每个 stage 通过 stride-2 操作（pool 或 strided conv）将空间尺寸减半，同时通道数翻倍。这保证了每层的计算量大致恒定。VGG 确立了这个范式，ResNet 直接继承。

原则 2：用小卷积核代替大卷积核。AlexNet 用 11×11 和 5×5，VGG 用全 3×3 证明了更好的效果。原因：同样的感受野下，多层小核的参数更少、非线性更多（你在上次提问中已经理解了这一点！）。

原则 3：用 1×1 瓶颈压缩计算。GoogLeNet 的 Inception Module 和 ResNet 的 Bottleneck Block 都用 1×1 conv 在高维和低维之间"搭桥"——在低维做重活，高维传信息。

原则 4：用 Global Average Pooling 替代 FC 层。GoogLeNet 首创，后来成为标配。消除了 FC 层的参数爆炸问题，同时让网络对输入尺寸不敏感。

Lecture 11 将介绍更多现代架构（ResNeXt、SENet、EfficientNet、NFNet 等），但它们全部都是建立在 Lecture 8 这四个模型的思想之上。更远的未来，当你在 Lecture 17-18 学到 Vision Transformer (ViT) 时，你会发现：

ViT 的 patch embedding 就是一个大 stride 的卷积（类似 GoogLeNet 的 aggressive stem）。ViT 的每个 Transformer block 都有残差连接（直接来自 ResNet）。ViT 的 FFN 层就是两个 1×1 卷积（先升维再降维的 bottleneck 结构）。所以 Lecture 8 的内容不是"历史课"——它是理解现代一切深度网络的基础。
