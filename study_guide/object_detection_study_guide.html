核心挑战：为什么检测比分类难得多？

Object Detection 面临三大挑战：(1) 可变数量的输出——一张图可能有 1 个物体也可能有 50 个，输出维度不固定；(2) 两种输出类型——既要预测"是什么"（分类），又要预测"在哪里"（回归）；(3) 大图像——分类通常用 224×224，检测需要 ~800×600 才能看到小物体。

PART 02

Transfer Learning：检测的基石

L13 Slides 12–29（WI2022 新增）

WI2022 版本在进入检测之前，先花了大量篇幅讲 Transfer Learning——因为 几乎所有现代检测器都依赖在 ImageNet 上预训练的 backbone。

Feature Extraction（特征提取）

冻结预训练 CNN 的所有参数，只训练最后一层线性分类器。速度快、数据需求少，但精度有上限。

Fine-Tuning（微调）

用预训练权重初始化整个网络，然后在新任务上继续训练所有层。需要更多数据和计算，但精度更高。

Trick：用更小的学习率（原始的 1/10），先做 Feature Extraction 再 Fine-Tune，BN 层用 test mode。

Architecture Matters（Slide 23–24）

Justin 展示了一组惊人的数据：COCO 检测 mAP 从 DPM（Pre-DL 时代）的 5，到 Fast R-CNN + AlexNet 的 15，到 Faster R-CNN + ResNet-50 的 29，到 Mask R-CNN + ResNeXt-152 的 46。几乎所有提升都来自更好的 backbone 架构——同一个检测框架，换一个更强的 backbone，性能就大幅提升。这就是为什么 Transfer Learning 如此重要。

PART 03

Object Detection 任务定义

L13 Slides 32–38 / FA2019 Slides 8–16

输入：一张 RGB 图像。输出：一个检测集合——每个检测包含一个类别标签（从预定义类别集中选）和一个边界框（x, y, w, h 四个数）。

Bounding Box 的表示方式

边界框通常是轴对齐的（axis-aligned），用 (x, y, w, h) 或 (x₁, y₁, x₂, y₂) 表示。WI2022 还区分了 Modal Detection（只框可见部分）和 Amodal Detection（框整个物体包括被遮挡的部分）——实际中绝大多数数据集用 modal detection。

从单物体到多物体：关键的跳跃

单物体检测很简单——加两个 head（分类 head 用 softmax loss，回归 head 用 L2 loss），做 Multitask Learning。但多物体检测需要输出可变数量的框——这导致了整个检测架构设计的复杂性。

为什么不能用滑动窗口？

一个直觉的想法是"在图像上滑动一个窗口，每个位置用 CNN 分类"。但一张 800×600 的图有 ~5800 万个可能的框（所有位置×所有大小的组合）！即使每个框只需 1ms，也要 16 小时处理一张图。所以我们需要 Region Proposals——先用快速方法找到~2000 个"可能包含物体"的候选框。

PART 04

评估基础：IoU / NMS / mAP

L13 Slides 39–50 / L14 Slides 98–109 / FA2019 Slides 35–57

检测器的评估需要三个关键工具。让我们逐一讲解。

① Intersection over Union（IoU）

IoU（Jaccard Similarity）
IoU = Area(Intersection) / Area(Union)
IoU > 0.5 = "decent"，IoU > 0.7 = "pretty good"，IoU > 0.9 = "almost perfect"

IoU 衡量两个框的重叠程度，范围从 0（完全不重叠）到 1（完全重合）。它是检测评估的原子操作——后续的 NMS 和 mAP 都以它为基础。

② Non-Max Suppression（NMS）

检测器通常在同一个物体附近产生多个重叠的框。NMS 用贪心策略去除冗余：

1
选择分数最高的框保留它
2
计算该框与所有剩余框的 IoU删除 IoU > 阈值（如 0.7）的框
3
重复直到没有剩余框
NMS 的局限 当物体高度重叠（如人群场景），NMS 可能误删正确的检测框——这是一个至今没有完美解决方案的问题。

③ Mean Average Precision（mAP）

mAP 是检测器最标准的评估指标。完整流程：

  1. 对所有测试图运行检测器（带 NMS）
  2. 对每个类别，按分数从高到低排序所有检测框：
    • 每个检测框与 GT 比较——若 IoU > 阈值且该 GT 尚未被匹配，标为 TP（True Positive）
    • 否则标为 FP（False Positive）
    • 逐步计算 Precision 和 Recall，画出 PR 曲线
  3. AP = PR 曲线下面积（衡量单个类别的检测质量）
  4. mAP = 所有类别 AP 的平均值
  5. COCO mAP = 在 IoU 阈值 0.5, 0.55, ..., 0.95 上分别计算 mAP，再取平均——更严格
如何获得 AP = 1.0？ 命中所有 GT 框（IoU > 阈值），且没有任何 FP 排在 TP 前面——即所有预测都是正确的，且没有漏检。
PART 05

R-CNN：朴素但奠基性的方案

L13 Slides 51–67 / FA2019 Slides 27–34

R-CNN（Region-based CNN，Girshick et al. 2014）是第一个将深度学习应用于目标检测的里程碑工作。

1
生成 Region Proposals（~2000 个）使用 Selective Search 等启发式方法
2
Resize 每个 region 到 224×224直接 warp 到固定大小
3
对每个 region 独立跑一遍 CNN提取特征并分类 + 回归 bbox transform
4
NMS 去重输出最终检测结果

Bounding Box Regression 的参数化（Slide 33）

R-CNN 不直接预测框坐标，而是预测一个 transform——从 proposal box 到 GT box 的变换：

Box Transform
bx = px + pw · tx   by = py + ph · ty
bw = pw · exp(tw)   bh = ph · exp(th)
p = proposal box, t = 预测的 transform, b = 输出 box。平移用相对值，缩放用 log-space——确保缩放始终为正。
R-CNN 的致命问题：慢！ 2000 个 region × 每个跑一遍完整的 CNN = ~2000 次前向传播。这些 region 大量重叠，导致巨大的重复计算。R-CNN 处理一张图需要 ~47 秒——完全无法实际部署。
PART 06

Fast R-CNN：共享计算

L14 Slides 10–48 / FA2019 Slides 58–70

Fast R-CNN（Girshick, 2015）的核心改进极其精妙：把 CNN 从"每个 region 独立跑"变成"整张图只跑一次"。

Fast R-CNN：Backbone 只跑一次（蓝色），然后从共享的 Feature Map 上裁剪每个 RoI 的特征（紫色），再用轻量级网络分类 + 回归
为什么这么快？ R-CNN：2000 个 region × 完整 CNN = 2000 次前向传播。Fast R-CNN：1 次完整 CNN + 2000 次轻量级 per-region head。大部分计算在 backbone 中发生，且只做一次——所有 region 共享同一个 feature map。
PART 07

RoI Pool 与 RoI Align

L14 Slides 30–47 / FA2019 Slides 71–89

"从特征图上裁剪一个 region"看似简单，实际上有微妙的工程细节。这里有两种方案：

RoI Pool（Fast R-CNN, 2015）

  1. 将 proposal 坐标投影到特征图上
  2. Snap 到最近的网格单元（取整）
  3. 将 snapped 区域划分为 H×W 的子区域（如 7×7）
  4. 在每个子区域内做 max pooling

问题：Snap 导致轻微错位（misalignment）；不同大小的 region 被划分成大小不一的子区域，不优雅。

RoI Align（Mask R-CNN, He et al. 2017）

  1. 将 proposal 坐标投影到特征图上——不 snap，保留浮点坐标
  2. 将区域划分为 H×W 的等大子区域（坐标可能不在网格上）
  3. 在每个子区域内均匀采样点，用双线性插值（bilinear interpolation）计算每个采样点的特征
  4. 在每个子区域内做 max pooling

双线性插值公式：f(x,y) = Σ_{i,j} f(i,j) · max(0, 1-|x-i|) · max(0, 1-|y-j|)

关键优势：完全可微分！梯度可以流回到四个相邻网格点。而且消除了 snap 导致的错位问题。

RoI PoolRoI Align
坐标处理Snap 到网格（取整）保留浮点坐标
采样方式直接取网格值双线性插值
可微分性部分（max pool 可微，snap 不可微）完全可微分
精度有错位无错位
实际使用已被淘汰标准做法
PART 08

Faster R-CNN：学习 Region Proposals

L14 Slides 51–72 / FA2019 Slides 94–103

Fast R-CNN 把瓶颈从 CNN 前向传播转移到了 Region Proposal 上——Selective Search 在 CPU 上运行，成了新的速度瓶颈。Faster R-CNN 的解决方案：用神经网络学习 proposals。

Region Proposal Network（RPN）

RPN 在 backbone 的特征图上工作。核心思想是Anchor Box——在特征图的每个位置放置 K 个预定义大小/比例的框，对每个框预测两件事：

是否包含物体？

Binary classification（object / not object）。输出形状：2K × H × W

Box Transform

从 anchor 到真实框的变换。输出形状：4K × H × W

Anchor 的正负标签分配（Slides 65–68）

类型条件训练中的作用
PositiveIoU ≥ 0.7（与某个 GT 框）
或：与某个 GT 框 IoU 最高的那个		训练分类 + 回归
NegativeIoU < 0.3（与所有 GT 框）只训练分类，不训练回归
Neutral0.3 ≤ IoU < 0.7完全忽略，不参与训练

Faster R-CNN 的四个 Loss

RPN 分类

anchor 是否包含物体

RPN 回归

anchor → proposal 的 transform

Detection 分类

proposal 的具体类别

Detection 回归

proposal → final box 的 transform

Two-Stage 架构总结

Faster R-CNN 是一个 Two-Stage 检测器。第一阶段（per-image）：backbone 提取特征 + RPN 生成 proposals。第二阶段（per-region）：RoI Align 裁剪特征 + 分类 + bbox 回归。这种"先粗筛再精分"的策略在精度上长期领先。

PART 09

Feature Pyramid Network（FPN）

L14 Slides 73–83（WI2022 新增）

检测中的一个核心挑战是尺度变化——同一张图中可能同时有很大和很小的物体。CNN 的高层特征分辨率低（适合检测大物体），低层特征分辨率高但语义弱（适合检测小物体）。FPN 的解决方案：自顶向下融合。

FPN：每一层 P_i 既有高层语义信息（通过 top-down 路径），又有低层的空间细节（通过 lateral connection）
FPN 为什么有效？

低层特征（Stage 2–3）空间分辨率高但语义弱——它"看得清"小物体但"不懂"它们是什么。高层特征（Stage 4–5）语义强但分辨率低。FPN 通过 top-down 路径把高层的语义信息传递到低层，让每个尺度都同时拥有高分辨率和强语义——这就是它在小物体检测上效果显著的原因。

PART 10

Single-Stage 检测器

L14 Slides 84–97 / FA2019 Slides 104–106

Two-stage 检测器（Faster R-CNN）精度高但速度受限于第二阶段的 per-region 处理。Single-stage 检测器直接从特征图预测类别和框，跳过第二阶段。

RetinaNet（Lin et al., ICCV 2017）

和 RPN 的结构几乎一样，但不再是"object / not object"的二分类，而是直接做 C+1 类分类（C 个物体类别 + 1 个背景类别）。

核心问题：类别不均衡——绝大多数 anchor 是背景（negative），真正包含物体的 anchor 很少。大量 easy negative 会主导 loss。

解决方案：Focal Loss——对 easy examples 降低 loss 权重，让模型更关注 hard examples。这让 single-stage 检测器首次在精度上接近 two-stage。

实际中 RetinaNet 也使用 FPN 处理多尺度。

FCOS（Tian et al., ICCV 2019）——Anchor-Free

FCOS 完全不使用 anchor box，而是对特征图上的每个点直接预测：

  1. 类别分数：该点是否落在某个 GT 框内？如果是，属于哪个类别？ → C × H × W
  2. 到框边界的距离：预测到 GT 框的上、下、左、右四个边的距离 (L, R, T, B) → 4 × H × W
  3. Centerness：衡量该点离框中心有多近 → 1 × H × W
Centerness Score
centerness = √(min(L,R)/max(L,R) · min(T,B)/max(T,B))
框中心 = 1，框边缘 = 0。用于抑制远离中心的低质量检测。

测试时每个点的最终置信度 = class score × centerness。FCOS 也使用 FPN 处理多尺度。
