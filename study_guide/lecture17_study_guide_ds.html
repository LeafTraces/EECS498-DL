PART 01

全局定位：从 RNN 到 Attention 的进化

对应 Slides 4–11

在 Lecture 16 中我们掌握了 RNN 及其变体（LSTM/GRU），它们通过隐藏状态逐步阅读序列，但始终将所有历史信息压缩进一个固定长度的向量。当序列变长（如 T=1000），这个 “信息瓶颈” 会导致严重的信息丢失与梯度问题。

Lecture 16 → Lecture 17 的知识跃迁 RNN 的核心矛盾：我们必须把整个输入序列放进一个上下文向量 c（通常 c = hT），然后解码器只能看着这一个向量生成整个输出。
Attention 的解决方案：让解码器在每一个时间步都能“回头看”编码器的所有隐藏状态，并决定关注哪些部分。
从 “压缩所有信息到一个向量” 变为 “按需检索输入的不同位置” —— 这是本课最核心的思维转变。
PART 02

Seq2Seq 的信息瓶颈：Attention 的诞生动机

对应 Slides 5–12
x1→ h1→ h2→ h3→ h4
↓ c = hT ↓
y1← s1(c)← y2← s2(c)
传统 Seq2Seq：整个输入被压缩为一个上下文向量 c，解码器每个时间步都只能“盲看”这一个向量。
瓶颈的量化理解 当 T = 1000 时，hT 的维度可能只有 512 或 1024。这意味着我们要用几百个浮点数去概括一部小说或一篇长文档的全部语义。
信息论上，这必然导致早期时间步的信息被“遗忘”或“扭曲”——即使 LSTM 也无法完全解决，因为固定大小的向量最终会饱和。
PART 03

Bahdanau Attention：动态对齐

对应 Slides 13–24

Bahdanau et al. (ICLR 2015) 提出了第一个实用的 Attention 机制。核心思想：在解码器的每个时间步 t，计算一个新的上下文向量 ct，它是编码器所有隐藏状态 hi 的加权和。

对齐分数：et,i = fatt(st−1, hi)    (fatt 是一个 MLP)
注意力权重：at,i = softmax(et,:)i    (0 < at,i < 1, Σ at,i = 1)
上下文向量：ct = Σi at,i hi
解码器状态更新：st = gU(yt−1, st−1, ct)
一切都是可微的！ 我们不直接监督注意力权重 at,i —— 模型通过梯度下降自己学会“对齐”。梯度可以流畅地从解码器损失流回对齐网络 fatt 再到编码器。这是端到端学习的典范。

可视化注意力权重（Slides 21–24）

在英语→法语翻译中，注意力矩阵常呈对角线分布（词序对应），但也会自动发现语序差异（如形容词后置）和动词变位。

Attention 不关心顺序 解码器将编码器的隐藏状态视为无序集合 {hi}，仅通过相似度来“检索”。这一特性为后续的 Self-Attention 和 Transformer 铺平了道路。
PART 04

Attention 的一般形式：Query-Key-Value

对应 Slides 44–51

我们可以将 Attention 抽象为一个通用的软检索操作：给定一个查询 q，在一组键值对 (ki, vi) 中，根据查询与键的相似度，加权聚合值向量。

Query q → similarity(q, ki) → softmax → weights ai
↓
Output y = Σ ai vi
Bahdanau Attention通用 Attention (QKV)
Query = st−1 可学习查询矩阵 WQ
Keys = hi（直接用） Keys = XWK（输入通过 Key 投影）
Values = hi（直接用） Values = XWV（输入通过 Value 投影）
相似度 = MLP(st−1, hi) 相似度 = 点积 / 缩放点积

为什么要分离 Key 和 Value？

Key 负责计算相似度，Value 负责提供聚合内容。这允许模型使用不同特征空间分别优化“如何匹配”和“匹配后取什么信息”。在实践中，WK 和 WV 独立学习，极大地提升了灵活性。

PART 05

缩放点积注意力（Scaled Dot-Product Attention）

对应 Slides 45–48
相似度：ei = (q · ki) / √dk
权重：a = softmax(e)
输出：y = Σ ai vi
为什么需要除以 √dk ？ 点积的期望值随维度 dk 增大而增大。较大的点积会导致 softmax 进入饱和区（梯度接近 0），训练极慢。除以 √dk 保持方差为 1，保证 softmax 的平滑性。

批量矩阵运算

若同时有 NQ 个查询和 NX 个输入向量，可一次性计算：

Q (NQ × dk), K = XWK (NX × dk), V = XWV (NX × dv)
E = Q KT / √dk    (NQ × NX)
A = softmax(E, dim=1)    (行方向 softmax)
Y = A V    (NQ × dv)

这完全由矩阵乘法实现，可充分利用 GPU 并行性。

PART 06

自注意力（Self-Attention）与排列不变性

对应 Slides 56–70

在 Self-Attention 中，查询 Q、键 K 和值 V 都来自同一个输入序列 X。每个位置都去“关注”序列中所有其他位置（包括自己）。

Q = X WQ,   K = X WK,   V = X WV
Yi = Σj softmax( Qi · Kj / √dk )j Vj

排列不变性（Permutation Invariance）

Self-Attention 将输入视为集合而不是序列。如果你打乱输入向量的顺序，输出顺序也会相应打乱，但值不变。这意味着模型本身不知道词序！

必须注入位置信息 纯粹的 Self-Attention 无法区分“狗咬人”和“人咬狗”。要获得序列感知能力，必须添加位置编码（Positional Encoding） —— 无论是固定的正弦编码还是可学习的位置嵌入。
虽然 Justin 在本课 slides 仅一笔带过，但这正是 Transformer 的关键一步。

CNN with Self-Attention（Slides 82–87）

Self-Attention 也可用于图像的像素/特征图。例如将 H×W 的特征图展平为 NX = H·W 个向量，让每个位置关注所有位置，从而捕获全局上下文。SAGAN (ICML 2018) 是早期成功案例。

PART 07

Masked Self-Attention 与自回归生成

对应 Slides 72–73

在语言建模（预测下一词）这类自回归任务中，位置 i 只能看到前面的词（位置 1…i），不能“偷看”未来。我们通过在 softmax 前将未来位置的相似度设为 -∞ 来实现。

E'i,j = Ei,j   if j ≤ i   else   -∞
A = softmax(E', dim=1)
PyTorch 实现方式 创建一个上三角矩阵，需要屏蔽的位置填充 -inf。
mask = torch.triu(torch.ones(T, T), diagonal=1).bool()
scores.masked_fill_(mask, float('-inf'))
PART 08

多头自注意力（Multi-Head Self-Attention）

对应 Slides 74–81

与其用一个高维的注意力，不如将 Q、K、V 切割成 H 个较小的“头”，每个头独立进行注意力计算，然后将所有头的输出拼接并线性投影。

headh = Attention( XWQh, XWKh, XWVh )
MultiHead(X) = Concat( head1, …, headH ) WO

单头 512 维

所有信息在一个空间中混合，可能被平均效应掩盖细粒度模式。

8 个头 × 64 维

不同头可学会关注不同子空间：一个头专注语法结构，另一个关注语义相关，第三个寻找指代关系。并行计算，总计算量相似。

Transformer 论文中的超参数 Transformer-Base 使用 H=8, dk=dv=64, dmodel=512。
PART 09

Transformer 架构详解

对应 Slides 92–102

Transformer 完全抛弃了 RNN，仅用 Attention + MLP + 残差连接 + LayerNorm 堆叠而成。每个块的结构为：

xi ↓ Multi-Head Self-Attention ↓ + (残差连接) ↓ LayerNorm ↓ MLP (两层, ReLU) ↓ + (残差连接) ↓ LayerNorm ↓ 输出
Transformer Block：自注意力是向量之间唯一的交互，LayerNorm 和 MLP 独立应用于每个位置。

关键设计选择

  • 残差连接：帮助梯度流动，可训练深层网络（12–24 层）。
  • LayerNorm：在每个向量内部做归一化，稳定训练。
  • MLP：通常两层，第一层升维（4×），第二层降维。每个位置独立处理，但完全相同。
  • 位置编码：在输入词嵌入上加上位置信息（此处必须补充，slides 虽未展开但实际架构必备）。
Pre-Norm vs Post-Norm 原论文将 LayerNorm 放在残差之后（Post-Norm），但现在许多实现采用 Pre-Norm（LayerNorm 在 Attention/MLP 之前），训练更稳定，成为默认选择。
PART 10

序列建模三范式对比与 Transformer 的可扩展性

对应 Slides 88–91
模型长程依赖并行度内存
RNN (LSTM) 好（理论 O(1) 步可见全部） 差（必须顺序计算） 低（O(T)）
1D CNN 差（需要多层堆叠） 好（并行） 低（O(T)）
Self-Attention 很好（单层全局交互） 好（并行） 高（O(T2)）

Transformer 用 Self-Attention 解决了长程依赖与并行化的矛盾，但代价是二次复杂度的内存消耗。这也是后续 Longformer、Linformer、BigBird 等高效 Transformer 变体要解决的核心问题。

PART 11

预训练与微调：NLP 的 ImageNet Moment

对应 Slides 103–121

Transformer 的巨大规模使其能在海量无标注文本上预训练，然后在下游任务微调。这被 Justin 称为 “NLP 的 ImageNet Moment”。

预训练（Pretraining）

下载海量文本，训练一个巨型 Transformer 做语言建模（预测下一个词）或掩码语言建模（BERT 的 cloze 任务）。

微调（Finetuning）

在预训练模型之上添加小分类头，用极少任务数据更新全部或部分参数，获得 SOTA 结果。

Scaling Law（Slides 104–112）

从 65M 的 Transformer-Base 到 175B 的 GPT-3，再到 280B 的 Gopher，模型参数、数据量和训练成本呈指数增长。GPT-3 甚至展示了强大的上下文学习（in-context learning）能力——仅通过 prompt 示例就能执行翻译、语法纠错、文学风格模仿等任务，无需梯度更新。

Vibe Coding 时代的启示 当你用 AI 编程时，你实际上在“prompt”一个类似 GPT-3 的模型。理解 Attention 和 Transformer 的架构，能让你更精准地设计 prompt，也更能理解 AI 输出的局限性（例如幻觉源于注意力权重的错误分配）。
PART 12

工程实现：从零搭建 Scaled Dot-Product Attention

PyTorch 核心代码

下面给出一个可运行的、经过优化的多头自注意力模块，结合了前面所有理论细节。

import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads, dropout=0.0):
        super().__init__()
        assert d_model % num_heads == 0, "d_model must be divisible by num_heads"
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads

        # 合并 QKV 投影为一个大矩阵，提高效率
        self.W_qkv = nn.Linear(d_model, 3 * d_model)
        self.W_o = nn.Linear(d_model, d_model)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, mask=None):
        # x: (batch, seq_len, d_model)
        B, T, C = x.shape

        # 线性投影并分割 Q,K,V
        qkv = self.W_qkv(x)  # (B, T, 3*C)
        q, k, v = qkv.chunk(3, dim=-1)  # each (B, T, C)

        # 重塑为多头: (B, num_heads, T, d_k)
        q = q.view(B, T, self.num_heads, self.d_k).transpose(1, 2)
        k = k.view(B, T, self.num_heads, self.d_k).transpose(1, 2)
        v = v.view(B, T, self.num_heads, self.d_k).transpose(1, 2)

        # 缩放点积注意力
        scale = math.sqrt(self.d_k)
        attn_scores = (q @ k.transpose(-2, -1)) / scale  # (B, H, T, T)

        if mask is not None:
            attn_scores = attn_scores.masked_fill(mask == 0, float('-inf'))

        attn_weights = F.softmax(attn_scores, dim=-1)
        attn_weights = self.dropout(attn_weights)

        # 加权聚合 Value
        out = attn_weights @ v  # (B, H, T, d_k)
        out = out.transpose(1, 2).contiguous().view(B, T, C)

        # 输出投影
        return self.W_o(out)
代码中的关键设计
  • 将 Q、K、V 的投影合并为一个 nn.Linear(d_model, 3*d_model)，减少模块调用开销。
  • 使用 .view() 和 .transpose() 巧妙实现多头拆分，而无需循环。
  • 缩放因子取 math.sqrt(d_k)，符合理论推导。
  • mask 支持填充屏蔽（pad mask）和因果屏蔽（look-ahead mask）。

场景：你需要为超长文档（100万 token）设计一个分类模型 普通 Transformer 的内存复杂度是 O(T²)，无法直接使用。你需要了解高效 Attention 变体（稀疏 Attention、线性 Attention、FlashAttention）。而这些变体都源自今天所学的 QKV 框架。掌握基础才能驾驭前沿。
