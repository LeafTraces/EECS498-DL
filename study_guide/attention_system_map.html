MAP 01 全景鸟瞰：一条推导链

Lecture 17 的所有内容可以被还原为一条连续的逻辑推导链。每一步都由前一步暴露的问题驱动，不存在凭空出现的设计——理解了这条链，你就拥有了重新推导 Transformer 的能力。

起点

Seq2Seq 的信息瓶颈

问题：整个输入序列被压缩成一个固定维度的向量 c = h_T，长序列信息严重丢失

编码器必须把所有信息塞进一个瓶颈向量，解码器从这个单一向量出发生成——信息在瓶颈处被大量丢弃。

v0

Seq2Seq Attention（Bahdanau 2015）

方案：让解码器在每一步动态"回头看"编码器的所有隐藏状态
e_t,i = f_att(s_{t-1}, h_i) // MLP 计算相似度 a_t = softmax(e_t) // 归一化为概率 c_t = Σ_i a_t,i · h_i // 加权求和

解决 信息瓶颈——每步都能访问所有输入   遗留 f_att 是 MLP，不通用，不高效

v1

点积 Attention

问题：MLP 相似度计算慢，无法矩阵化
e_i = q · X_i // 点积：最简相似度

解决 效率——变成矩阵乘法   遗留 高维下点积值爆炸

v2

缩放点积 Attention

问题：D=512 时点积方差为 512，softmax 饱和
e_i = q · X_i / √D // 方差归一化到 1

解决 数值稳定性   遗留 只有一个 query

v3

多 Query 并行

问题：多个位置需要同时做 attention，逐个计算浪费并行性
E = Q X^T / √D // (N_Q × N_X) 一次算完

解决 并行性   遗留 X 身兼"匹配"和"检索"两职

v4

Key-Value 分离

问题："用什么来匹配"和"匹配到了取什么"是两个不同的概念
K = X W_K, V = X W_V E = Q K^T / √D, A = softmax(E), Y = A V

解决 关注点分离——独立学习索引和内容   遗留 Q 还是来自外部

SA

Self-Attention

问题：序列理解需要内部词与词之间互相参考，但 Q 来自外部
Q = X W_Q, K = X W_K, V = X W_V // Q, K, V 全部从同一个 X 投影而来

解决 序列自我交互   遗留 O(N²) 内存 + 丢失位置信息 + 单一注意力模式

MHA

Multi-Head Attention

问题：一个 softmax 只能表达一种关注模式，语言关系是多维的
Split D → H heads of D/H each Y = Concat(head_1, ..., head_H) W_O

解决 多种关系并行捕捉   遗留 还缺少非线性变换和归一化

TFM

Transformer Block

方案：MHA + Residual + LayerNorm + MLP = 一个完整的处理单元，堆叠 L 层
x = x + MHA(LN(x)) // 全局交互 x = x + MLP(LN(x)) // 逐向量非线性变换

最终形态 Attention Is All You Need

MAP 02 统一数学框架

所有 Attention 变体共享同一个数学骨架。下面把 Cross-Attention、Self-Attention、Masked Self-Attention、Multi-Head Self-Attention 统一在一个框架下——它们的区别仅在于 Q 的来源、是否有 mask、head 数量三个参数。

统一 Attention 公式
Q = XQ WQ ,   K = XKV WK ,   V = XKV WV
E = Q KT / √Dhead + M
A = softmax(E, dim=-1)
Y = A V
其中 M 是 mask 矩阵（无 mask 时为全零矩阵），Dhead = D / H

所有变体通过三个"开关"控制：

变体XQ 来源XKV 来源Mask MHeads H
Cross-Attention解码器序列编码器序列全零1
Self-Attention同一个 X同一个 X全零1
Masked Self-Attn同一个 X同一个 X上三角 = -∞1
MH Self-Attn同一个 X同一个 X全零 / -∞H (如 8)
统一视角的价值

这个统一框架说明 Attention 的本质只有一个操作："在一个向量集合中，根据 query 做软检索"。Cross vs Self 只是"谁提问"的区别；Mask 只是"能看到哪些位置"的约束；Multi-Head 只是"同时进行多少次独立检索"的选择。理解了这一点，任何新的 Attention 变体（Flash Attention, GQA, MLA 等）你都能快速定位它改了统一框架中的哪一部分。

MAP 03 Attention 的四种形态

基于统一框架，Lecture 17 涉及的 Attention 可以分为四种形态。它们在不同的应用场景中出现：

CROSS Cross-Attention

场景：一个序列查询另一个序列

应用：Seq2Seq 翻译（解码器看编码器）、图像描述（RNN 看 CNN 特征图）、Transformer Decoder 中间层

Q ≠ KV 来源。两组向量之间的交互。

SELF Self-Attention

场景：序列理解自身（每个位置看所有位置）

应用：BERT 编码器、ViT 编码器、序列表示学习

Q = KV 来源。序列内部所有位置全连接交互。

MASKED Masked Self-Attention

场景：自回归生成（只能看已经生成的部分）

应用：GPT 系列、语言生成、Transformer Decoder 第一层

E 的上三角填 -∞，softmax 后权重为 0。保证因果性。

MULTI-HEAD Multi-Head Attention

场景：需要同时捕捉多种关系模式

应用：所有现代 Transformer（正交于上面三种，可和任何一种组合使用）

Split D → H 个子空间，每个子空间独立做 attention。参数量不变。

分类关系

前三种（Cross / Self / Masked Self）是关于 "谁看谁" 的分类，互斥。Multi-Head 是关于 "同时看几遍" 的正交维度，可以和前三种任意组合。实际中几乎总是使用 Multi-Head 版本。

MAP 04 Attention 的五个核心性质

无论是哪种 Attention 变体，它们都共享一些基本性质。理解这些性质是理解 Transformer 架构决策的关键。

① 完全可微分（End-to-End Differentiable）

Attention 的所有操作——矩阵乘法、除以常数、softmax、加权求和——都是可微分的。因此注意力权重不需要人工监督，模型通过反向传播自动学习"在哪里看"。

工程含义：你可以把 Attention 层插入任何计算图中，梯度会自然流过。不需要额外的损失函数来训练注意力权重。

② 置换等变性（Permutation Equivariance）

Self-Attention 满足 f(permute(X)) = permute(f(X))。打乱输入顺序，输出也被相应打乱——模型天然不知道位置信息。

直接后果：必须引入 Positional Encoding。这不是可选的附件，而是 Transformer 能处理序列的必要条件。

深层含义：Self-Attention 的操作对象本质上是无序集合而非有序序列。RNN 天然处理序列（有内在的时间步），Self-Attention 天然处理集合（位置信息需要外加）。

③ O(N²) 时空复杂度

注意力矩阵 E ∈ ℝN×N，计算 QKT 需要 O(N²D) 运算，存储 E 和 A 需要 O(N²) 内存。

序列长度 NE 矩阵大小内存（float32, 单层单 head）
512262,144~1 MB
4,09616,777,216~64 MB
32,7681,073,741,824~4 GB

工程含义：这就是 Flash Attention、滑动窗口 Attention（Swin）、稀疏 Attention 等优化技术存在的原因——都是在 O(N²) 的基本限制下做文章。

④ Attention 核心运算无可学习参数

一个容易被忽视的事实：E = QKT/√D → A = softmax(E) → Y = AV 这三步中没有任何可学习参数。所有参数都在 WQ, WK, WV, WO 这四个投影矩阵中。

深层含义：Attention 本身是一个固定的路由机制——它只决定"信息从哪来到哪去"。投影矩阵决定"什么样的信息参与路由"和"路由后的信息长什么样"。

⑤ 本质是集合上的操作

Attention 的输出 y_i = Σ_j a_{i,j} · V_j 是 V 的加权和——这个操作和 V 的顺序无关（加法交换律）。Attention 把输入视为一个无序集合，而非有序序列或空间网格。

与 RNN/CNN 的根本区别：

  • RNN：操作对象是有序序列（h_t 依赖 h_{t-1}）
  • CNN：操作对象是空间网格（卷积核在空间上滑动）
  • Attention：操作对象是向量集合（每个元素和所有其他元素交互）
MAP 05 Transformer Block 的设计原理

单独的 Multi-Head Attention 还不够——它只提供了"全局交互"能力，还需要"局部变换"（MLP）和训练稳定性保障（Residual + LayerNorm）。Transformer Block 是这四个组件的精密组合。

设计哲学

Transformer Block 只有一个地方让向量们互相交流（MHA），其余操作全是逐向量独立的。这种"全局交互 + 局部变换"的交替设计有两个好处：(1) 可并行性——除了 MHA 之外的所有操作都可以逐向量独立计算；(2) 可解释性——信息流动的唯一通道就是 attention 权重矩阵 A，你可以直接可视化它来理解模型在做什么。

四个组件各自的角色

组件角色操作范围如果去掉会怎样
Multi-Head Self-Attention向量间信息交换全局（所有 N 个向量）向量永远看不到彼此，无法建模任何关系
MLP逐向量非线性变换局部（每个向量独立）只有线性变换，表达能力严重不足

MAP 06 三大范式对比：RNN vs CNN vs Attention

Lecture 17 的一个核心贡献是明确了 Self-Attention 相对于 RNN 和 CNN 的优势——为什么 Transformer 最终"赢了"。

RNN

操作对象：有序序列

✓ 天然有序

✓ 可处理变长

✗ 无法并行

✗ 长程信息衰减

1D CNN

操作对象：空间网格

✓ 高度可并行

✓ 局部模式高效

✗ 受限感受野

✗ 需要堆叠很多层

Self-Attention

操作对象：向量集合

✓ 高度可并行

✓ 一层全局交互

✗ O(N²) 内存

✗ 需要位置编码

Transformer 胜出的根本原因

Transformer 的优势不仅仅是"效果好"——它赢在可扩展性（scalability）。RNN 无法并行，成为训练速度的瓶颈；CNN 需要堆很多层才能获得全局视野，参数效率低。Self-Attention 既可并行又天然全局交互，让"堆更多数据 + 堆更多参数 = 更好的性能"这条 scaling law 成为可能。从 65M 参数的 Transformer-Base 到 175B 参数的 GPT-3，架构本身几乎没有变——变的只是层数、维度和训练数据。

MAP 07 张量形状速查表

面试和作业中最常犯的错误是张量形状搞混。以下是带 batch 维度的完整形状参考（B = batch, N = 序列长度, D = 模型维度, H = head 数）：

变量形状含义
X（输入）(B, N, D)B 个样本，每个 N 个 token，每个 D 维
WQ, WK(D, D)投影到 Q/K 空间（包含所有 head）
WV(D, D)投影到 V 空间
Q, K, V（投影后）(B, N, D)和 X 同形状
Q, K, V（reshape 后）(B, H, N, D/H)拆成 H 个 head
E = QKT/√d(B, H, N, N)每个 head 的注意力分数矩阵
M (mask)(1, 1, N, N) 或 (N, N)广播到所有 batch 和 head
A = softmax(E+M)(B, H, N, N)每行是概率分布
Yheads = AV(B, H, N, D/H)每个 head 的输出
Yconcat(B, N, D)reshape 合并所有 head
Y = YconcatWO(B, N, D)最终输出，和输入同形状
关键观察

输入和输出形状完全相同：(B, N, D) → (B, N, D)。这就是为什么 Transformer Block 可以任意堆叠——每一层的输出可以直接作为下一层的输入，加上 residual connection 还能保证梯度流通。

MAP 08 完整代码骨架

最后，把整个 Transformer Block 的代码从底层到顶层完整呈现。这是你在 A3 中实现过的结构的理论对应。

import torch
import torch.nn as nn
import math

# ─── Layer 1: Scaled Dot-Product Attention ───
def scaled_dot_product_attention(Q, K, V, mask=None):
    # Q: (B, H, N_Q, D_head)
    # K: (B, H, N_KV, D_head)
    # V: (B, H, N_KV, D_V)
    d = Q.size(-1)
    E = Q @ K.transpose(-2, -1) / math.sqrt(d)  # (B,H,N_Q,N_KV)
    if mask is not None:
        E = E.masked_fill(mask == 0, float('-inf'))
    A = torch.softmax(E, dim=-1)                 # (B,H,N_Q,N_KV)
    Y = A @ V                                     # (B,H,N_Q,D_V)
    return Y, A

# ─── Layer 2: Multi-Head Attention ───
class MultiHeadAttention(nn.Module):
    def __init__(self, D, H):
        super().__init__()
        self.H, self.d_head = H, D // H
        self.W_Q = nn.Linear(D, D)
        self.W_K = nn.Linear(D, D)
        self.W_V = nn.Linear(D, D)
        self.W_O = nn.Linear(D, D)

    def forward(self, X_Q, X_KV, mask=None):
        B, N, D = X_Q.shape
        # Project + reshape into (B, H, N, d_head)
        Q = self.W_Q(X_Q).view(B, -1, self.H, self.d_head).transpose(1,2)
        K = self.W_K(X_KV).view(B, -1, self.H, self.d_head).transpose(1,2)
        V = self.W_V(X_KV).view(B, -1, self.H, self.d_head).transpose(1,2)
        # Attention
        Y, A = scaled_dot_product_attention(Q, K, V, mask)
        # Concat + project
        Y = Y.transpose(1,2).reshape(B, -1, D)   # (B, N, D)
        return self.W_O(Y)

# ─── Layer 3: Transformer Block (Pre-Norm) ───
class TransformerBlock(nn.Module):
    def __init__(self, D, H, D_ff=None):
        super().__init__()
        D_ff = D_ff or 4 * D
        self.ln1 = nn.LayerNorm(D)
        self.attn = MultiHeadAttention(D, H)
        self.ln2 = nn.LayerNorm(D)
        self.mlp = nn.Sequential(
            nn.Linear(D, D_ff), nn.GELU(), nn.Linear(D_ff, D)
        )

    def forward(self, x, mask=None):
        # Sub-layer 1: Self-Attention with residual
        x = x + self.attn(self.ln1(x), self.ln1(x), mask)
        # Sub-layer 2: MLP with residual
        x = x + self.mlp(self.ln2(x))
        return x      # (B, N, D) → (B, N, D)  形状不变！
代码与理论的对应

注意 TransformerBlock.forward 只有两行核心逻辑——一行 attention，一行 MLP，每行都有 residual 和 LayerNorm。从 65M 参数到 175B 参数的模型，这两行代码的结构完全一致——改变的只是 D、H、D_ff 和层数 L 的值。整个 Transformer 革命建立在这两行代码的数万次堆叠之上。

最终的自检：如果你能用一句话准确描述每个概念，你就真正掌握了 Lecture 17。

概念一句话
