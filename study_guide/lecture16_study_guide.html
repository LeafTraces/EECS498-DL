PART 02

顺序处理非顺序数据：RNN 的独特视角

对应 Slides 11–13

Justin 用三张 slides 展示了 RNN 一个极为优雅的用途：将非序列数据（如图像）当作序列来处理。这打破了"RNN 只能处理序列"的刻板印象。

Visual Attention（ICLR 2015）

通过一系列"glimpses"（瞥视）来分类图像。模型每次只看图像的一小部分，逐步积累信息，最终做出分类决策。

类比：你不需要看清整幅画——先扫一眼轮廓，再聚焦关键细节。

DRAW（ICML 2015）

逐块生成图像——每个时间步在画布上添加新的内容，就像人类画家一笔一笔地作画。

Slide 13 还展示了与油画模拟器的集成——每个时间步输出一个新的笔画。

核心洞见 RNN 的本质不是"处理序列数据"，而是"通过内部状态累积信息"。只要你能把问题定义为"逐步积累信息"的过程，RNN 就能派上用场——无论数据本身是不是序列。这个洞见在 2025 年的扩散模型（逐步去噪生成图像）中依然熠熠生辉。
PART 03

Vanilla RNN：核心公式与隐藏状态

对应 Slides 14–22

这是整堂课最重要的公式。Vanilla RNN（也称 Elman RNN，以 Jeffrey Elman 教授命名）的核心是一个简单的循环关系：

ht = tanh(Whh ht−1 + Wxh xt + bh)
yt = Why ht + by
公式拆解 ht：当前时间步的隐藏状态（hidden state）——RNN 的"记忆"
ht−1：上一时间步的隐藏状态
xt：当前时间步的输入
Whh：隐藏状态到隐藏状态的权重矩阵（核心！跨时间传递信息）
Wxh：输入到隐藏状态的权重矩阵
tanh：非线性激活函数（将值压缩到 [−1, 1]）

RNN 计算图展开（Slide 19–22）

RNN 的核心是一个重复使用的计算单元 fW，在时间维度上展开：

h0 → fW → h1 → fW → h2 → fW → h3
每次使用相同的权重 W——这就是"循环"的含义
为什么叫"Vanilla"？ "Vanilla RNN"就是最朴素的 RNN——只有一个隐藏状态向量 h，用一个 tanh 来更新。它简单优雅，但后面你会看到它在长序列上有严重的梯度问题。LSTM 和 GRU 就是为了解决这些问题而生的"增强版"。
PART 04

RNN 计算图的四种模式

对应 Slides 23–27

根据输入和输出的序列长度关系，RNN 有四种经典的计算图模式。理解这些模式是设计 RNN 架构的基础。

模式结构典型任务Slide
Many-to-Many 每个输入 xt 对应一个输出 yt 词性标注、逐帧视频分类 23–24
Many-to-One 读入整个序列，只在最后输出一个 y 情感分析、视频分类 26
One-to-Many 单个输入，产生整个输出序列 图像描述生成、音乐生成 27
Many-to-Many（异步） 先读完整个输入（编码），再生成整个输出（解码） 机器翻译 = Seq2Seq 28–29
实践记忆法 输入序列长度 = 决定"读"的模式；输出序列长度 = 决定"写"的模式。Many-to-One 就是"读很多，写一个"；One-to-Many 就是"读一个，写很多"。Seq2Seq 就是"先读完（Many-to-One），再开始写（One-to-Many）"。
PART 05

Seq2Seq：编码器-解码器架构

对应 Slides 28–29

Seq2Seq（Sutskever et al., NIPS 2014）是 Many-to-One + One-to-Many 的组合，是机器翻译、文本摘要等任务的经典架构。

x1→ fW→ h1→ fW→ h2→ fW→ h3→ fW→ hT
↑ Encoder（Many-to-One）：将整个输入序列压缩成一个向量 hT ↓
hT→ y1→ fW2→ y2→ fW2→ ...
↑ Decoder（One-to-Many）：从 hT 出发逐步生成输出序列
Seq2Seq 的核心瓶颈 整个输入序列被压缩成单个固定长度的向量 hT。对于长序列（如长句子），这个"信息瓶颈"会导致性能下降。Attention 机制（Lecture 17）正是为解决这个问题而提出的——它让解码器可以直接"看到"编码器的所有中间隐藏状态，而不只是最后一个。

语言建模实战：字符级 RNN

对应 Slides 30–54

Justin 用大量 slides 展示了字符级语言建模——这是一个完美的 RNN 教学示例，因为它的效果肉眼可见。

任务定义（Slides 30–36）

给定前 t−1 个字符，预测第 t 个字符。训练序列 "hello"，词汇表 [h, e, l, o]：

"h"→ RNN→ 预测 "e"
"he"→ RNN→ 预测 "l"
"hel"→ RNN→ 预测 "l"
测试时：采样一个字符，将其反馈给模型，生成下一个字符，循环往复。

min-char-rnn.py：112 行 Python（Slide 48）

Andrej Karpathy 的 min-char-rnn.py 是字符级 RNN 的极简实现——仅 112 行 Python。它完美展示了 RNN 的核心：前向传播、反向传播、参数更新。

训练过程的文本演化（Slides 50–54）

这是一个极其震撼的可视化：你眼睁睁地看着模型从完全随机的乱码，逐步学会单词拼写、语法结构、甚至莎士比亚的风格：

阶段生成文本示例
初始 tyntd-iafhatawiaoihrdemot lytdws e ,tfti, astai f...
训练一会 "Tmont thithey" fomesscerliund Keushey. Thom here sheulke...
更久 Aftair fall unsuch that the hall for Prince Velzonski's...
充分训练 可生成莎士比亚风格的十四行诗（Slide 49）、Linux 内核代码（Slides 59–61）、甚至代数几何教科书（Slide 55）

嵌入层（Slide 42）

输入字符用 one-hot 向量 表示。与权重矩阵相乘等价于"查表"（lookup）——提取权重矩阵的某一列。实践中，这一操作被抽象为 Embedding Layer（nn.Embedding），它本质上就是一个可学习的查找表。

PyTorch 中的字符级语言模型核心代码 
# 极简字符级 RNN 语言模型
import torch
import torch.nn as nn

class CharRNN(nn.Module):
    def __init__(self, vocab_size, hidden_size, num_layers=1):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, hidden_size)
        self.rnn = nn.RNN(hidden_size, hidden_size, num_layers, batch_first=True)
        self.fc = nn.Linear(hidden_size, vocab_size)

    def forward(self, x, h=None):
        x = self.embed(x)          # (B, T) -> (B, T, H)
        out, h = self.rnn(x, h)       # out: (B, T, H), h: (L, B, H)
        logits = self.fc(out)         # (B, T, H) -> (B, T, V)
        return logits, h
PART 07

Truncated BPTT：截断反向传播通过时间

对应 Slides 43–47

训练 RNN 需要Backpropagation Through Time（BPTT）——在展开的计算图上做反向传播。但对于非常长的序列（如整本书），完整的 BPTT 在计算上不可行。Truncated BPTT 是工程上的核心解决方案。

完整 BPTT

前向传播遍历整个序列，然后反向传播遍历整个序列。

问题：序列长度 = 100 万时，内存和时间都不可接受。梯度还会爆炸/消失。

Truncated BPTT

将序列分成块（chunks），在每个块内做 BPTT。隐藏状态跨块保持，但梯度只在块内传播。

效果：隐藏状态可以"永远"向前传递（保持长期记忆），但反向传播只限于较小步数。

Truncated BPTT 的核心权衡
截断长度 ↑ → 更好的长期依赖学习，但计算成本 ↑ 且梯度问题加剧
截断长度 ↓ → 更快的训练，但模型只能学到较短范围的依赖
工程实践 典型的截断长度是 20–50 个时间步。在 PyTorch 中，你可以通过 hidden.detach() 来切断梯度流，实现 Truncated BPTT。这是个关键的工程技巧——在 AI 生成的代码中经常缺失，需要你手动补充。

可解释隐藏单元：RNN 学到了什么

对应 Slides 62–68

这是 Justin 本人参与的研究（Karpathy, Johnson, and Fei-Fei, ICLR Workshop 2016）。他们在训练好的字符级 RNN 中发现了功能高度特化的隐藏单元——这些单元不是被设计出来的，而是自发涌现的。

引文检测单元
“”
Quote Detection Cell
在引号内激活，引号外关闭——像一个"引号开关"
行长度跟踪单元
↔
Line Length Tracking Cell
跟踪当前行还剩多少字符到达行尾
If 语句单元
if
If Statement Cell
在 C 代码的 if 语句块内激活，块外关闭
注释单元

为什么这很重要？ 这项研究证明了 RNN 不是黑箱——至少部分隐藏单元具有清晰可解释的功能。在 2025 年，这种"机械可解释性"（Mechanistic Interpretability）研究在大语言模型上正如火如荼。理解 RNN 的可解释性是理解更大模型内部机制的第一步。这也提醒我们：好的模型自然会学习结构化的内部表示。
PART 09

图像描述：CNN + RNN 多模态融合

对应 Slides 69–80

Image Captioning 是一个经典的多模态任务：输入一张图像，输出描述该图像的文本序列。这需要将 CNN 的视觉理解与 RNN 的语言生成结合起来。

架构（Slides 70–78）

核心修改：在 RNN 的隐藏状态更新中注入图像特征向量 v（来自预训练 CNN 的最后一层）：

原始 RNN：ht = tanh(Whhht−1 + Wxhxt + bh)

修改后：ht = tanh(Whhht−1 + Wxhxt + Wihv + bh)

图像特征 v 在每个时间步都被使用——这告诉 RNN "你在描述的是什么图像"。

效果展示（Slides 79–80）

成功案例

  • "A cat sitting on a suitcase on the floor"
  • "Two giraffes standing in a grassy field"
  • "A tennis player in action on the court"

失败案例

  • "A woman is holding a cat in her hand"（实际可能是其他物体）
  • "A person holding a computer mouse on a desk"（误判物体）
2015 年的 Image Captioning 五连发 Slide 69 列出了 CVPR 2015 上五篇同时发表的图像描述论文——这标志着深度学习在多模态领域的爆发。核心思路惊人地一致：CNN 编码图像 + RNN 解码文本。这种"编码器-解码器"范式至今仍是多模态 AI 的基础。
PART 10

Vanilla RNN 的梯度流危机：爆炸与消失

对应 Slides 81–86

这是理解 为什么需要 LSTM 的关键。Vanilla RNN 在长序列上训练困难，根本原因在于梯度流的不稳定性。

问题根源（Slides 81–82）

计算 h0 的梯度需要经过多次与 W 相乘（实际上是 WhhT 的多次乘法）：

∂L/∂h0 ∝ (WhhT)t × (其他项)
条件现象后果
W 的最大奇异值 > 1 梯度爆炸 梯度变为 NaN/Inf，训练崩溃
W 的最大奇异值 < 1 梯度消失 早期时间步的梯度趋近于 0，模型无法学习长期依赖

解决方案（Slides 85–86）

应对梯度爆炸

梯度裁剪（Gradient Clipping）：如果梯度的 L2 范数超过阈值，就将其缩放到阈值大小。

grad_norm = torch.norm(grad)
if grad_norm > threshold:
    grad *= threshold / grad_norm

应对梯度消失

改变 RNN 架构！梯度裁剪治不了消失（梯度本身就很小，裁剪没有意义）。需要用加性交互（additive interactions）来改善梯度流——这就是 LSTM 的动机。

Vibe Coding 时代的诊断能力 当你用 AI 生成的 RNN 代码训练时，如果 loss 突然变成 NaN——这是梯度爆炸的典型症状。你需要知道加梯度裁剪，而不是盲目调参。如果 loss 下降极慢或完全不下降——可能是梯度消失，你需要考虑换 LSTM 或 GRU。这些诊断能力是 AI 无法替你做的。
PART 11

LSTM 深度解剖：四门控 + Cell State

对应 Slides 87–96

LSTM（Long Short Term Memory, Hochreiter & Schmidhuber, 1997）是解决梯度消失问题的经典方案。它的核心创新是引入了一个独立的 Cell State（细胞状态）ct，其更新方式是加性的而非乘性的。

LSTM 完整公式

四个门控的计算（一次矩阵乘法同时算出四个门）：
  [it; ft; ot; gt] = [σ; σ; σ; tanh](W [ht−1; xt] + bh)

Cell State 更新（加性！）：
  ct = ft ⊙ ct−1 + it ⊙ gt

隐藏状态输出：
  ht = ot ⊙ tanh(ct)

四门控的功能（Slide 91）

Input Gate
it
σ → [0,1]
控制是否写入新信息到 cell state
Forget Gate
ft
σ → [0,1]
控制是否擦除旧的 cell state 信息
Output Gate
ot
σ → [0,1]
控制暴露多少 cell state 给外部
Gate Gate
gt
tanh → [−1,1]
候选值——要写入什么新信息

LSTM 的无中断梯度流（Slides 93–95）

从 ct 到 ct−1 的反向传播只涉及逐元素乘法（ft ⊙），没有矩阵乘法！这意味着：

∂ct/∂ct−1 = ft   （逐元素！每个维度独立！）

这就是 LSTM 解决梯度消失的核心：梯度可以通过 cell state 几乎无损地跨时间步传播。

LSTM 与 ResNet 的血缘关系（Slide 96） LSTM 的 cell state 更新 ct = ft ⊙ ct−1 + it ⊙ gt 与 ResNet 的残差连接 y = F(x) + x 有着完全相同的核心思想：通过加性交互为梯度提供一条"高速公路"。Highway Networks（Srivastava et al., 2015）是二者之间的直接桥梁。这是深度学习史上最优雅的"殊途同归"之一。
PART 12

多层 RNN 与 GRU 等变体

对应 Slides 97–103

多层 RNN（Slides 97–99）

与 CNN 一样，RNN 也可以堆叠多层来增加模型容量。第 l 层的隐藏状态 htl 依赖于：

  • 同一层上一时间步的隐藏状态 ht−1l
  • 下一层同一时间步的隐藏状态 htl−1

在 PyTorch 中，只需设置 nn.LSTM(hidden_size, hidden_size, num_layers=3) 即可。

GRU：LSTM 的精简版（Slide 100）

Gated Recurrent Unit（Cho et al., 2014）将 LSTM 的四个门精简为两个门（reset gate rt 和 update gate zt），并将 cell state 和 hidden state 合并为一个。

LSTM

  • 4 个门：i, f, o, g
  • 2 个状态：ct（cell）+ ht（hidden）
  • 参数量较大
  • 表现稳定，工业界首选

GRU

  • 2 个门：r（reset）, z（update）
  • 1 个状态：ht（合并了 cell 和 hidden）
  • 参数量约减少 25%
  • 小数据集上有时优于 LSTM

进化搜索发现的变体（Slide 101）

Jozefowicz et al.（ICML 2015）用进化算法搜索了 10000 种 RNN 架构，发现了一些"奇怪但有效"的变体——有些使用 sign 函数而非 sigmoid，有些有非标准的门控连接。但 LSTM 和 GRU 仍然是最可靠的选择。

Justin 的总结（Slide 103）

本讲核心要点
  1. RNN 架构设计灵活——Many-to-One、One-to-Many、Seq2Seq 适应各种任务
  2. Vanilla RNN 简单但不实用——梯度爆炸/消失使长序列训练困难
  3. LSTM/GRU 是实际标准——加性交互（additive interactions）改善梯度流
  4. 梯度爆炸用梯度裁剪控制；梯度消失用 LSTM/GRU 的加性设计缓解
  5. 更好的架构和理解是当前研究热点——Attention（下节课！）是下一步进化

COMPETITIVE EDGE

Vibe Coding 时代的竞争力

场景：AI 生成了一个 LSTM 训练代码，但 loss 在几个 epoch 后变成 NaN AI 生成的代码通常遗漏梯度裁剪。你需要立即识别这是梯度爆炸问题，并在 optimizer 之后添加 torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=5.0)。这个诊断能力来自对 Part 10 的深刻理解——知道"为什么"比知道"怎么做"更重要。
场景：你需要为一个长文档分类任务选择模型 AI 可能建议"用 BERT"——但如果你的文档长度是 10000+ tokens，BERT 的 O(n²) 复杂度会让你破产。理解 RNN 的 O(n) 递推特性意味着你知道何时 LSTM 仍然是更好的选择。在资源受限或序列极长的场景下，LSTM 的递推效率无可替代。
场景：面试中被问到"LSTM 为什么能解决梯度消失" 标准答案涉及 cell state 的加性更新和逐元素门控。但真正有竞争力的回答会进一步指出：这本质上与 ResNet 的 skip connection 是同构的——都通过为梯度提供一条"无乘法的路径"来解决深度/时间上的梯度衰减。展示这种跨架构的洞察力，是区分"会用 API"和"真正理解"的关键。
