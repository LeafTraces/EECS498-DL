PART 02

rnn_step_forward & rnn_step_backward

函数：rnn_step_forward(), rnn_step_backward()

最底层的积木——单个时间步的 Vanilla RNN。

Vanilla RNN 单步公式
next_h = tanh(x · Wx + prev_h · Wh + b)
输入 x 和前一步隐藏状态 prev_h 各自做线性变换，加起来过 tanh

维度验证

x(N, D)当前时间步的输入
Wx(D, H)输入到隐藏的权重
prev_h(N, H)上一步的隐藏状态
Wh(H, H)隐藏到隐藏的权重
b(H,)偏置
next_h(N, H)当前步的隐藏状态（输出）

Forward 实现

# rnn_step_forward
a = x @ Wx + prev_h @ Wh + b       # (N, H)
next_h = torch.tanh(a)              # (N, H)
cache = (x, prev_h, Wx, Wh, next_h) # 保存反向传播需要的值
为什么用 tanh 而不是 ReLU RNN 的隐藏状态会在时间步之间反复相乘。ReLU 输出 [0, ∞)，反复相乘容易爆炸到无穷大。tanh 输出 (−1, 1)，相乘后数值被天然限制在有限范围内（虽然仍然会衰减——这就是 RNN 梯度消失的原因，也是 LSTM 被发明的动机）。

Backward 推导

反向传播的目标是从 dnext_h 回推 dx, dprev_h, dWx, dWh, db。分两步走：

第一步：穿过 tanh。 tanh 的导数是 1 − tanh²(x)，而 cache 中的 next_h 就是 tanh(a)，所以：

da = dnext_h · (1 − next_h²)   形状 (N, H)

第二步：穿过线性层 a = x @ Wx + prev_h @ Wh + b。 这和 A2 中 scores = X @ W + b 的反向传播完全一样：

# rnn_step_backward
x, prev_h, Wx, Wh, next_h = cache
da = dnext_h * (1 - next_h ** 2)    # tanh 的导数

dx     = da @ Wx.t()                 # (N, D)
dprev_h = da @ Wh.t()               # (N, H)
dWx    = x.t() @ da                  # (D, H)
dWh    = prev_h.t() @ da            # (H, H)
db     = da.sum(dim=0)               # (H,)
PART 03

rnn_forward & rnn_backward（整序列）

函数：rnn_forward(), rnn_backward()

把单步 RNN 循环 T 次，处理整个序列。

rnn_forward

输入 x 形状 (N, T, D)，循环 T 次调用 rnn_step_forward，每步的 prev_h 是上一步的 next_h：

N, T, D = x.shape
H = h0.shape[1]
h = torch.zeros(N, T, H, dtype=x.dtype, device=x.device)
cache = []
prev_h = h0

for t in range(T):
    prev_h, step_cache = rnn_step_forward(x[:, t, :], prev_h, Wx, Wh, b)
    h[:, t, :] = prev_h
    cache.append(step_cache)

rnn_backward — ⭐ 本节难点

反向传播需要注意一个关键细节：

梯度的两个来源

dh[:, t, :] 是时间步 t 从 loss 直接收到的梯度。

但时间步 t 还会通过 prev_h → next_h 的连接向 t+1 传递信息，所以还有来自下一个时间步回传的 dprev_h。

两部分必须加在一起！

N, T, H = dh.shape
D = cache[0][0].shape[1]

dx = torch.zeros(N, T, D, dtype=dh.dtype, device=dh.device)
dWx = torch.zeros_like(cache[0][2])   # (D, H)
dWh = torch.zeros_like(cache[0][3])   # (H, H)
db  = torch.zeros(H, dtype=dh.dtype, device=dh.device)
dprev_h = torch.zeros(N, H, dtype=dh.dtype, device=dh.device)

for t in range(T - 1, -1, -1):        # 从 T-1 到 0 反向遍历
    # 当前步总梯度 = loss 直接梯度 + 下一步回传的梯度
    dcur = dh[:, t, :] + dprev_h

    dx_t, dprev_h, dWx_t, dWh_t, db_t = rnn_step_backward(dcur, cache[t])

    dx[:, t, :] = dx_t
    dWx += dWx_t     # 累加！所有时间步共享同一组权重
    dWh += dWh_t
    db  += db_t

dh0 = dprev_h        # 最后剩下的 dprev_h 就是对 h0 的梯度
为什么 dWx、dWh、db 是累加的 RNN 的所有时间步共享同一组权重（这是 RNN "参数共享"的核心特点）。所以 W 的总梯度 = 每个时间步贡献的梯度之和。这就像你在 A2 中 SVM naive 版本里对 dW 做 += 的逻辑。
PART 04

WordEmbedding & temporal_softmax_loss

类：WordEmbedding  |  函数：temporal_softmax_loss()

WordEmbedding：一行代码

W_embed 是 (V, D) 的矩阵，每一行是一个单词的向量表示。x 是 (N, T) 的整数索引。输出就是用索引从矩阵中"查表"取出对应的行：

out = self.W_embed[x]    # PyTorch fancy indexing，自动处理 (N, T) → (N, T, D)

temporal_softmax_loss：也是一行

这是序列版本的交叉熵损失。x 形状 (N, T, V) 是每个时间步对词汇表的分数，y 形状 (N, T) 是真实词索引。题目要求 sum over T, mean over N，并忽略 <NULL> padding：

# F.cross_entropy 期望输入 (batch, C)，所以 reshape 一下
loss = F.cross_entropy(
    x.reshape(-1, x.shape[2]),   # (N*T, V)
    y.reshape(-1),               # (N*T,)
    ignore_index=ignore_index,
    reduction='sum'
) / x.shape[0]                   # sum over T, mean over N
为什么用 reduction='sum' 再除以 N 题目要求 "summing the loss over all timesteps and averaging across the minibatch"。如果用 reduction='mean'，它会除以有效 token 总数（排除 ignore_index 之后），这和题目要求的"只除以 N"不同。所以用 sum 再手动除以 batch size N。
PART 05

LSTM：step_forward & forward

类：LSTM  →  step_forward(), forward()

LSTM（Long Short-Term Memory）通过"门控机制"解决 Vanilla RNN 的梯度消失问题。核心区别是多了一个 cell state c——一条信息"高速公路"，让重要信息可以长距离传递而不衰减。

LSTM 的四个门

所有门的激活值在一次矩阵乘法中计算，然后切成四份：

a = x @ Wx + prev_h @ Wh + b     # (N, 4H) ← 注意是 4H 不是 H！
ai, af, ao, ag = a.chunk(4, dim=1) # 各 (N, H)
门公式含义
i 输入门 i = sigmoid(ai) 哪些新信息要写入 cell
f 遗忘门 f = sigmoid(af) 哪些旧信息要从 cell 中保留
o 输出门 o = sigmoid(ao) 哪些 cell 信息要输出到 hidden state
g 候选值 g = tanh(ag) 要写入 cell 的新信息内容
Cell state & Hidden state 更新
next_c = f · prev_c + i · g   （遗忘旧的 + 写入新的）
next_h = o · tanh(next_c)   （输出经 tanh 压缩的 cell 信息）

step_forward 代码

a = x @ self.Wx + prev_h @ self.Wh + self.b    # (N, 4H)
ai, af, ao, ag = a.chunk(4, dim=1)               # 各 (N, H)

i = torch.sigmoid(ai)
f = torch.sigmoid(af)
o = torch.sigmoid(ao)
g = torch.tanh(ag)

next_c = f * prev_c + i * g
next_h = o * torch.tanh(next_c)

forward 代码

N, T, D = x.shape
H = h0.shape[1]
hn = torch.zeros(N, T, H, dtype=x.dtype, device=x.device)

prev_h = h0
prev_c = c0    # 已在外面初始化为 zeros_like(h0)

for t in range(T):
    prev_h, prev_c = self.step_forward(x[:, t, :], prev_h, prev_c)
    hn[:, t, :] = prev_h
LSTM 如何解决梯度消失 关键在 cell state 的更新公式：next_c = f * prev_c + i * g。遗忘门 f 接近 1 时，prev_c 可以几乎无损地传递到下一步。这条"高速公路"让梯度也能畅通回传——不需要像 Vanilla RNN 那样经过 tanh 的反复压缩。
PART 06

dot_product_attention & AttentionLSTM

函数：dot_product_attention()  |  类：AttentionLSTM

这是你在 Attention Is All You Need 精读中学到的 Scaled Dot-Product Attention 的实际应用——让 LSTM 在生成每个词时，"看向"图像中最相关的区域。

dot_product_attention

prev_h(N, H)LSTM 隐藏状态 = Query
A(N, H, 4, 4)CNN 特征图 = Key & Value
attn(N, H)注意力加权后的特征
attn_weights(N, 4, 4)注意力权重（可视化用）
N, H, D_a, _ = A.shape

# 1. 将 A 展平：(N, H, 4, 4) → (N, H, 16)
A_flat = A.reshape(N, H, -1)

# 2. 计算注意力分数：prev_h^T @ A → (N, 1, 16)
scores = torch.bmm(prev_h.unsqueeze(1), A_flat)   # (N, 1, 16)
scores = scores.squeeze(1) / (H ** 0.5)            # (N, 16) scaled

# 3. Softmax 归一化
attn_weights = F.softmax(scores, dim=1)             # (N, 16)

# 4. 加权求和：A @ weights → (N, H)
attn = torch.bmm(A_flat, attn_weights.unsqueeze(2)) # (N, H, 1)
attn = attn.squeeze(2)                              # (N, H)

# 5. reshape 回 (N, 4, 4) 用于可视化
attn_weights = attn_weights.reshape(N, D_a, D_a)
和论文的对应关系

论文公式：Attention(Q, K, V) = softmax(QKT/√dk)V

这里 Q = prev_h, K = V = A_flat。prev_h 是"我在找什么"，A 是"图像哪里有什么"。注意力权重告诉模型"生成当前这个词时应该看图像的哪个区域"。

AttentionLSTM.step_forward

和普通 LSTM 几乎一样，只是多了一个 attn 输入：

# 和 LSTM 的唯一区别：多加一个 attn @ Wattn
a = x @ self.Wx + prev_h @ self.Wh + attn @ self.Wattn + self.b
ai, af, ao, ag = a.chunk(4, dim=1)

i = torch.sigmoid(ai)
f = torch.sigmoid(af)
o = torch.sigmoid(ao)
g = torch.tanh(ag)

next_c = f * prev_c + i * g
next_h = o * torch.tanh(next_c)

AttentionLSTM.forward

N, T, D = x.shape
H = h0.shape[1]
hn = torch.zeros(N, T, H, dtype=x.dtype, device=x.device)

prev_h = h0
prev_c = c0

for t in range(T):
    attn, _ = dot_product_attention(prev_h, A)   # 每步重新算注意力
    prev_h, prev_c = self.step_forward(x[:, t, :], prev_h, prev_c, attn)
    hn[:, t, :] = prev_h
PART 07

CaptioningRNN：__init__, forward, sample

类：CaptioningRNN  →  __init__(), forward(), sample()

这是最顶层的"胶水"——把图像编码器、词嵌入、RNN/LSTM、输出投影全部串在一起。

__init__：初始化所有组件

# 图像编码器（预训练 CNN）
self.image_encoder = ImageEncoder(pretrained=image_encoder_pretrained)

# 词嵌入层
self.wordvec = WordEmbedding(vocab_size, wordvec_dim)

# 根据 cell_type 选择 RNN 类型
if cell_type == 'rnn':
    self.rnn = RNN(wordvec_dim, hidden_dim)
elif cell_type == 'lstm':
    self.rnn = LSTM(wordvec_dim, hidden_dim)
elif cell_type == 'attn':
    self.rnn = AttentionLSTM(wordvec_dim, hidden_dim)

# 输出投影：hidden state → 词汇表分数
self.output_proj = nn.Linear(hidden_dim, vocab_size)

# 特征投影：CNN 特征 → h0（或 Attention 的 A）
if cell_type == 'attn':
    self.feat_proj = nn.Linear(self.image_encoder.out_channels, hidden_dim)
else:
    self.feat_proj = nn.Linear(self.image_encoder.out_channels, hidden_dim)

forward：训练时前向传播

# (1) 编码图像
features = self.image_encoder(images)       # (N, C, 4, 4)

if self.cell_type == 'attn':
    # Attention: 保留空间维度，投影到 (N, H, 4, 4)
    N, C, H_f, W_f = features.shape
    A = self.feat_proj(features.permute(0, 2, 3, 1))  # (N,4,4,H)
    A = A.permute(0, 3, 1, 2)                          # (N,H,4,4)
else:
    # RNN/LSTM: 全局平均池化后投影到 h0
    pooled = features.mean(dim=(2, 3))     # (N, C)
    h0 = self.feat_proj(pooled)            # (N, H)

# (2) 词嵌入
word_vecs = self.wordvec(captions_in)       # (N, T, W)

# (3) RNN/LSTM
if self.cell_type == 'attn':
    hiddens = self.rnn(word_vecs, A)
else:
    hiddens = self.rnn(word_vecs, h0)       # (N, T, H)

# (4) 输出投影
scores = self.output_proj(hiddens)          # (N, T, V)

# (5) 计算损失
loss = temporal_softmax_loss(scores, captions_out,
                            ignore_index=self.ignore_index)

sample：测试时逐步生成

这是唯一不能并行的部分——每一步用上一步的预测结果作为输入。

sample 和 forward 的区别

forward（训练）：所有时间步并行处理。每步输入是真实标签（teacher forcing）。

sample（测试）：逐步串行生成。每步输入是上一步预测的词。需要用 step_forward 而不是 forward。

sample 的伪代码结构（需要根据 cell_type 区分 RNN/LSTM/Attention 的初始化和单步调用）：

# 编码图像 → h0（和 forward 一样）
features = self.image_encoder(images)

# 初始化根据 cell_type 不同
if self.cell_type == 'attn':
    A = ...                      # 投影特征
    prev_h = A.mean(dim=(2, 3))  # h0
    prev_c = prev_h              # c0
elif self.cell_type == 'lstm':
    prev_h = h0
    prev_c = torch.zeros_like(h0)
else:
    prev_h = h0

# 第一个输入词是 <START>
cur_word = torch.full((N,), self._start, dtype=torch.long, device=images.device)

for t in range(max_length):
    # (1) 嵌入当前词
    word_vec = self.wordvec(cur_word)        # (N, W)

    # (2) RNN/LSTM 单步
    if self.cell_type == 'rnn':
        prev_h = self.rnn.step_forward(word_vec, prev_h)
    elif self.cell_type == 'lstm':
        prev_h, prev_c = self.rnn.step_forward(word_vec, prev_h, prev_c)
    elif self.cell_type == 'attn':
        attn, attn_w = dot_product_attention(prev_h, A)
        prev_h, prev_c = self.rnn.step_forward(word_vec, prev_h, prev_c, attn)
        attn_weights_all[:, t] = attn_w

    # (3) 预测下一个词
    scores = self.output_proj(prev_h)        # (N, V)
    cur_word = scores.argmax(dim=1)          # (N,)

    # (4) 存入 captions
    captions[:, t] = cur_word
