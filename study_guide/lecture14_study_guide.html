PART 01

全局定位：这讲到底在回答什么问题

对应 Slides 5–6, 91

在前面的课程里，我们主要在做“训练更强的模型”。但一旦模型训练好，就会出现两个更深的问题：

  • 表示问题：中间层到底编码了什么？
  • 因果问题：模型为什么会给出这个预测？哪些像素或特征起了关键作用？
  • 反演问题：如果我们固定模型，反过来优化输入，会得到什么样的图像？

Lecture 14 把这些问题统一起来，告诉你：可解释性和生成并不是两套完全不同的主题，它们都建立在“网络是可微函数”这个事实之上。

这一讲几乎所有方法都可以放进同一个研究范式：固定网络，换一种方式观察它，或者直接对输入进行优化。
一句话总纲 Lecture 14 研究的不是“如何训练 CNN”，而是“如何理解 CNN 学到的表征，以及如何利用这些表征进行分析和生成”。
PART 02

表征分析：从第一层滤波器到最后层特征空间

对应 Slides 5–13

第一层可视化最容易理解

卷积核直接作用于 RGB 像素，因此第一层的权重本身就可以被当作小图像可视化。你通常会看到边缘、颜色对比、方向性纹理等模式，这与经典视觉中的 Gabor-like filter 十分相似。

第一层为什么容易解释
输入是 3 × H × W，卷积核是 C × 3 × k × k。每个卷积核直接与像素块做加权和，因此权重本身就是“在找什么局部图案”的线索。
越靠近输入层，解释越接近低级视觉特征；越靠近高层，解释越偏向语义组合。

高层滤波器直接看权重，意义就没那么大了

一旦进入深层，卷积核的输入不再是 RGB 图像，而是前一层的 feature map。此时一个卷积核是在组合“前一层学到的模式”，权重本身不再能直观地对应某种可见纹理。因此，后面就需要借助 激活可视化、最大激活 patch 或 反向优化 来理解。

最后层特征空间是语义分析的关键入口

在 AlexNet 一类网络里，分类器前的高维向量（如 FC7）常被当作图像的语义表征。把很多图像送进网络、收集这些特征后，可以做两类分析：

Local

最近邻检索

如果用像素空间做最近邻，往往只会找出颜色和布局相近的图；如果用高层特征空间做最近邻，找到的更可能是语义相似的物体或场景。

Global

降维可视化

用 PCA、t-SNE 等方法把高维表征投到二维后，可以观察类别聚类、混淆边界、子簇结构，帮助判断表征是否“按语义组织起来了”。

理论直觉 一个好的表征并不要求在像素空间中相近，而是要求在任务相关的语义空间中相近。CNN 训练本质上就是在学习这样一个“把原始像素拉直为语义几何”的映射。

把高层特征向量当作 embedding 后，最近邻检索可以回答一个非常实用的问题：模型认为哪些图像“语义相似”。这比只看分类精度更能暴露表示质量。

PCA 更偏全局线性结构，t-SNE 更偏局部邻域结构。它们都不是“真实的低维语义空间”，而是观察高维表示结构的窗口。因此要把它们当作分析工具，而不是当作严格几何真相。

选定某层某个通道，把大量图像送进网络，记录该通道激活最高的位置，再把对应原图 patch 裁出来。这样你能看到“这个神经元最喜欢什么局部模式”。这通常比直接看权重更有可解释性。

PART 03

显著性：哪些像素真正影响了预测

对应 Slides 14–19

方法一：Occlusion

最直接的方法是把图像的某一小块遮住，再看目标类别分数下降多少。如果遮住某块后分数大幅下降，就说明该区域对当前预测很重要。

Perturbation Occlusion Saliency

  1. 选择一个滑动窗口，把该区域像素置零或置均值。
  2. 重新前向传播，计算目标类别分数变化。
  3. 把分数下降幅度映射回对应位置，形成热力图。

它的优点是概念直观，缺点是计算代价大，而且对遮挡窗口大小、步长非常敏感。

方法二：Saliency via Backprop

如果模型是可微的，就可以直接计算目标类别分数对输入像素的梯度。直觉上，梯度大的像素表示：稍微改动这个像素，就会明显改变该类别得分。

Saliency 的核心量
给定类别 c 的未归一化分数 S_c(x)，计算 ∂S_c / ∂x；再对 RGB 通道取绝对值并聚合，得到二维 saliency map。
slide 特别强调用的是 unnormalized class score，也就是 Softmax 之前的 logit。
Occlusion

优点：黑盒友好

不需要梯度，模型不可微也能用；很适合做 sanity check。

Backprop

优点：快且精细

一次 backward 就能得到整张图的敏感性，但更依赖局部线性近似，解释上要谨慎。

可解释性中的一个常见误区 saliency map 告诉你的是“对分数敏感的方向”，不等于严格的因果解释。梯度大不一定说明该区域包含完整语义证据，也可能只是模型在该位置最容易被扰动。
工程实践 自己实现时，通常要把输入图像设置为 requires_grad=True，前向拿到目标类别 logit 后调用 backward()，再读 x.grad。这是 PyTorch autograd 最经典的教学例子之一。
PART 04

中间特征：guided backprop 与最大激活 patch

对应 Slides 20–23

对于中间层神经元，我们也可以问：它究竟在看图像中的哪种模式？

最大激活 Patch

它是数据驱动的“例子式解释”：看看在真实图像数据里，这个神经元最常被哪些 patch 激活。

优点是解释非常朴素，缺点是只能看到“真实数据中出现过的模式”。

Guided Backprop

它是梯度驱动的“敏感性解释”：固定某个中间神经元，对输入做反传，看看哪些像素最影响这个神经元。

slide 的关键改动是：在每个 ReLU 上，反向时只允许正梯度通过，因此结果通常更清晰。

Guided Backprop 的直觉
标准反传：ReLU 只根据前向激活是否 > 0 决定梯度能否通过。
Guided Backprop：再额外要求反向梯度也必须 > 0。
它让图更“好看”，但也意味着它不再是标准梯度，因此解释时要明确这是一种经过修改的可视化规则。
学习重点 最大激活 patch 和 guided backprop 经常被一起看，因为它们分别从数据统计视角和局部微分视角解释同一个神经元。二者相互印证时，解释通常更可信。
PART 05

统一框架：固定网络，对输入做梯度上升

对应 Slides 24–35

这一部分是整讲最重要的抽象飞跃。前面 saliency 是“对输入求梯度”，现在更进一步：直接把输入当成可优化变量。

统一优化目标
I* = arg max_I f(I) + R(I)
f(I) 是你希望大的神经元、类别分数或特征目标，R(I) 是让结果更自然的正则项。
1
固定网络参数。模型不再学习，只有输入图像在变。
2
定义目标函数。可以是某个类别分数、某个通道激活、某层特征、Gram loss 等。
3
反向传播到输入。用 autograd 计算目标对像素的梯度。
4
更新图像。做梯度上升或下降，并配合各种正则化或投影步骤。

从零图像或噪声开始，最大化某个类别分数，就能生成“网络心中的该类别典型输入”。这不是数据集里的真实图，而是网络偏好的可视化结果。

如果只做纯梯度上升，图像常常会长得很噪。slide 给出的改进包括：L2 正则、Gaussian blur、裁剪小像素值、裁剪小梯度值。这些都在引入“自然图像先验”。

一个神经元未必只表示单一视觉概念，而可能混合多种“面向”。multi-faceted visualization 的思路是：承认一个 neuron 有多种激活模式，并把这些模式分别可视化出来。

# 统一输入优化伪代码
I = init_image()  # zero / noise / real image

for t in range(num_steps):
    score = objective(model, I)
    reg = regularizer(I)
    loss = -(score + reg)   # 如果想做梯度上升，可写成负号转成最小化

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    project_or_clip(I)
一个非常值钱的抽象能力 如果你能把 saliency、class visualization、feature inversion、DeepDream、style transfer 都看成“固定网络上的输入优化”，你就真正抓住了这讲的统一思想。
PART 06

对抗样本、特征反演与 DeepDream

对应 Slides 36–45

Adversarial Examples

对抗样本说明：高维线性结构 + 局部梯度信息 足以让你沿着看似很小的扰动方向，把网络推向完全错误的类别。slide 里给出的版本是从任意图像出发，直接做梯度上升去骗过网络。

理论含义 对抗样本不是一个边角现象，而是提醒我们：判别模型学到的决策边界，可能在人类感知看来非常脆弱。这件事后来发展成了完整的对抗鲁棒性研究方向。

Feature Inversion

给定某层特征向量，找一张新图，使它在该层的表示尽量接近原图。这在问：这个特征层保留了哪些信息，又丢掉了哪些信息。

Feature Inversion 的目标
L(I) = || Φ(I) - Φ(I_target) ||² + λ R(I)
Φ 是固定 CNN 的某层特征，R(I) 常用 total variation 等图像先验。

如果从浅层反演，重建图通常更接近原图像素；如果从深层反演，重建图会更保留语义而丢失细节。这正反映了层级表征的抽象程度。

DeepDream

DeepDream 不再针对一个单独神经元，而是放大某层现有激活：网络已经“看到了什么”，就把它进一步夸张出来。于是天空里会长出狗脸、建筑物上会冒出鸟羽纹理。

DeepDream 的等价目标
I* = arg max_I Σ_i f_i(I)²
不是生成一个类别原型，而是放大当前图像中已经存在的高层模式。
Feature Inversion

目标：重建已有表征

问“给定表征，原图可能长什么样”。重点在信息保留与丢失。

DeepDream

目标：放大已有特征

问“网络已经看到了什么，把它继续增强会怎样”。重点在偏好和幻觉。

slide 提到的工程技巧 DeepDream 的经典实现会配合 jitter、L1 归一化梯度、裁剪像素值、多尺度处理。这些技巧的作用都是让优化更稳定、结果更连续、更少棋盘格或局部重复。
PART 07

纹理合成：Gram Matrix 为什么有效

对应 Slides 52–67

这一部分是从“分析表征”走向“用表征做生成”的关键桥梁。经典最近邻纹理合成按像素或局部 patch 复制纹理，而神经纹理合成则用 CNN 特征统计量来定义“什么叫同一种纹理”。

Gram Matrix 的构造
给定某层特征 F ∈ ℝ^(C × HW)，定义 G = F F^T，则 G ∈ ℝ^(C × C)。
Gram 矩阵记录的是通道之间的二阶相关性，可以理解为一种未归一化协方差统计。
Gram Matrix 抛弃了精确空间排列，却保留了通道共现统计，这正是“纹理感”最需要而“物体布局”最不需要的部分。

为什么它适合描述纹理

纹理更多由“局部模式出现了多少、彼此怎样共现”决定，而不太取决于它们在图像中绝对放在哪里。Gram Matrix 恰好保留了这种空间无关的统计结构。

Texture Neural Texture Synthesis Pipeline

  1. 用预训练 CNN（slide 中是 VGG-19）提取多层特征。
  2. 对每层特征计算 Gram Matrix。
  3. 从随机噪声初始化待生成图像。
  4. 让生成图在各层的 Gram Matrix 接近目标纹理图。
  5. 反向传播到图像并迭代更新。
层级含义 用较浅层做 Gram 匹配，更偏局部纹理；用较深层做 Gram 匹配，会恢复更大的模式和更复杂的结构。slide 66 展示的正是这种层级差异。
PART 08

风格迁移：内容损失 + 风格损失

对应 Slides 68–81

风格迁移把前面的两类目标组合起来：

  • 内容损失：生成图与内容图在某层特征上接近。
  • 风格损失：生成图与风格图在若干层的 Gram Matrix 上接近。
Style Transfer 总目标
L_total = λ_content L_content + λ_style L_style + λ_tv L_TV
其中 L_content 约束“画什么”，L_style 约束“怎么画”，L_TV 常用来鼓励图像平滑。

内容损失

通常在较深层计算：|| Φ_l(I_out) - Φ_l(I_content) ||²。因为深层特征更能保留语义结构，而不强制像素级一致。

风格损失

通常在多层计算：Σ_l || G_l(I_out) - G_l(I_style) ||²。多层能同时覆盖细颗粒纹理与大尺度笔触模式。

slide 75 很重要：如果内容权重大，输出更像原图，只染上一层风格；如果风格权重大，输出会更“画”，但内容结构可能被改得更厉害。这个平衡不是细节调参，而是任务定义的一部分。

slide 76 指出：把风格图缩放到不同大小后再提特征，会改变被匹配到的风格尺度。大图更强调大笔触/全局结构，小图更强调细局部纹理。

多风格混合可以通过对多个风格图的 Gram Matrix 加权平均实现。这说明“风格”在这个框架里不是离散标签，而是可在统计空间中连续插值的对象。

你应该抓住的本质 Neural Style Transfer 并不是“神秘地学会艺术”，而是把高层特征看作内容，把通道相关性统计看作风格，再用优化把二者拼在同一张图上。
PART 09

快速风格迁移与 Instance Normalization

对应 Slides 82–90

原始风格迁移每处理一张图，都要在 VGG 上做很多轮前向/反向优化，非常慢。Lecture 14 的解决方案非常经典：把优化过程蒸馏成一个前馈网络。

Fast Fast Neural Style Transfer

  1. 为某一种风格训练一个前馈生成网络。
  2. 训练时，仍然使用预训练 CNN 上定义的内容损失和风格损失。
  3. 训练好后，推理时只需一次前向传播就能完成风格化。

为什么 Instance Normalization 会变重要

slide 87–88 特别强调：Instance Normalization 最初就是在 style transfer 场景里被提出并证明有效的。直觉上，它会按每个样本、每个通道做归一化，更符合“不同图像风格强度本来就该单独调节”的需求。

BatchNorm

依赖 batch 统计

适合判别任务，但在风格迁移里会让不同样本之间互相耦合，风格控制不够干净。

InstanceNorm

按样本独立归一化

更容易“洗掉”原图整体对比度/风格偏差，让网络专注于重新注入目标风格。

One Network, Many Styles

再进一步，可以让一个网络支持多种风格：不同风格共享卷积主干，但拥有各自的归一化尺度与偏移参数，这就是 conditional instance normalization。slide 90 甚至展示了训练后风格之间的混合能力。

一个很通用的工程启发 很多慢优化方法都可以被“编译”为快前馈模型：先用精确但昂贵的优化目标定义任务，再训练一个网络去近似这个优化解。这种思路后来在超分辨率、图像恢复、扩散蒸馏等地方都反复出现。

工程实践与通用能力总结

结合 Summary 与工程视角扩展

把整讲压缩成一张能力地图

先看表示
第一层滤波器、最近邻、PCA / t-SNE、最大激活 patch，回答“模型把什么看成相似”。
再看梯度
Occlusion、saliency、guided backprop，回答“哪些输入最影响这个输出/神经元”。
最后反向做生成
Class visualization、对抗样本、feature inversion、DeepDream、texture synthesis、style transfer，回答“如果我让网络最满意，会得到什么输入”。

最推荐的自学顺序 如果你真想学扎实，建议按这个顺序亲手实现： saliency map → class visualization → feature inversion → Gram Matrix → style transfer loss。 这样你会一步步把“固定网络上的输入优化”这条主线真正打通。
