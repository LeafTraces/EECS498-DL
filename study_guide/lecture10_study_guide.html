PART 02

Sigmoid 与 Tanh：历史遗产与三大问题

对应 Slides 7–23

Justin 花了大量篇幅（Slides 10–22）讲 Sigmoid 的三个问题。这不是因为你以后会用 Sigmoid（你不会），而是因为理解这些问题本身就是理解所有激活函数设计的关键。

Sigmoid：σ(x) = 1 / (1 + e-x)

Sigmoid 将任意实数映射到 (0, 1)，曾经很流行，因为它可以被解释为神经元的"发射率"。但它有三个致命问题：

问题一：饱和区梯度消失（Saturated neurons kill gradients）

这是最严重的问题（Slide 22 明确标注"Worst problem in practice"）。

Sigmoid 的导数为 σ'(x) = σ(x)(1 - σ(x))。当输入 x 很大（如 x = 10）或很小（如 x = -10）时，σ(x) 接近 1 或 0，导数趋近于 0。在反向传播中，上游梯度要乘以这个局部梯度，如果局部梯度接近 0，那么往下游传递的梯度也接近 0——这就是梯度消失。

Slide 12 的三问三答

Justin 在计算图上画了一个 sigmoid gate，问了三个关键问题：

x = -10 时？ σ(-10) ≈ 0，梯度 ≈ 0 × (1-0) = 0 → 梯度为零，上游信号完全被杀死

x = 0 时？ σ(0) = 0.5，梯度 = 0.5 × 0.5 = 0.25 → 梯度最大，但只有 0.25（最好情况下也只有输入梯度的 1/4）

x = 10 时？ σ(10) ≈ 1，梯度 ≈ 1 × 0 = 0 → 梯度为零

这意味着即使在最理想的情况下（x=0），Sigmoid 也会将梯度衰减到 1/4。在深层网络中，经过多层 Sigmoid，梯度会指数级衰减到接近零，完全无法训练。

问题二：输出不是零中心的（Not zero-centered）

这是一个更微妙但同样重要的问题。Slides 15–19 给出了精彩的数学分析：

考虑第 ℓ 层的一个神经元：

hi(ℓ) = Σj wi,j(ℓ) σ(hj(ℓ-1)) + bi(ℓ)

对权重 wi,j 求梯度：

∂L/∂wi,j(ℓ) = σ(hj(ℓ-1)) × ∂L/∂hi(ℓ)

关键点：绿色部分是 Sigmoid 的输出，永远为正（在 0 到 1 之间）。因此 ∂L/∂wi,j 的符号完全由上游梯度 ∂L/∂hi 决定——同一层的所有权重梯度的符号都相同！

Slide 18 的几何直觉：锯齿形更新 如果所有权重的梯度符号相同，那意味着权重向量只能沿着"全正"或"全负"的方向更新——在二维空间中就是只能向右上或左下走。如果最优解在右下方向，你就只能走"锯齿形"路径，每一步先向右上走一点，再向左下修正。这让优化变得极其低效。

这就是为什么零中心的激活函数（如 Tanh、ReLU）更好——它们的输出有正有负，权重梯度可以自由地取任意符号组合。

问题三：exp() 计算开销

相比加法和取 max 操作，exp() 的计算成本更高。这在现代 GPU 上不是主要问题，但也是一个小缺点。

Tanh：改进但不够

Tanh 将数值映射到 (-1, 1)，是零中心的，解决了问题二。但它仍然会在大值和小值处饱和，梯度消失问题依然存在。所以 Tanh 只是一个过渡方案。

Sigmoid

输出范围：(0, 1)

零中心：否

饱和：两端都饱和

实践建议：不要使用

Tanh

输出范围：(-1, 1)

零中心：是

饱和：两端都饱和

实践建议：不要使用（除非特殊场景）

PART 03

ReLU 家族：从 ReLU 到 GELU

对应 Slides 24–37

ReLU: f(x) = max(0, x)

ReLU（Rectified Linear Unit）是深度学习的游戏改变者。它的优势非常明确（Slide 24）：

  • 正区间不饱和：x > 0 时，导数恒为 1，梯度可以无衰减地传回去
  • 计算极其高效：只需要一个 max 操作，比 exp() 快得多
  • 收敛速度快：实践中比 Sigmoid/Tanh 快约 6 倍（AlexNet 论文中的实验）

但 ReLU 也有两个问题：

问题一：不是零中心的——输出总是 ≥ 0，和 Sigmoid 有同样的锯齿形更新问题（但严重程度较轻）。

问题二：Dead ReLU（死亡 ReLU）——这是 ReLU 最令人头疼的问题。

Dead ReLU 问题（Slides 27–29）

当 x < 0 时，ReLU 的输出为 0，梯度也为 0。如果一个 ReLU 神经元的输入始终为负（比如因为权重初始化不当，或者学习率过大导致权重更新后输入变为负值），那么这个神经元将永远输出 0，梯度也永远为 0——它"死了"，再也不会被更新。

Slide 28 用一个直观的图展示了这个现象：数据点云（data cloud）在某些 ReLU 的"死区"（x < 0 那一侧），这些 ReLU 永远不会被激活。

缓解办法：用略微正的偏置初始化（如 b = 0.01），让初始时大部分神经元都处于激活状态。但这只是一个 heuristic，不完美。

Leaky ReLU: f(x) = max(αx, x)

Leaky ReLU 在 x < 0 时给一个小斜率 α（通常 α = 0.01 或 0.1），彻底解决了 Dead ReLU 问题——负区间也有梯度，神经元不会"死"。

PReLU（Parametric ReLU）：让 α 成为可学习的参数，由网络自己决定负区间的斜率。

ELU 与 SELU

ELU（Exponential Linear Unit）在负区间使用 α(ex - 1)，输出更接近零均值，负区间有饱和特性（可以增加对噪声的鲁棒性），但需要计算 exp()。

SELU（Scaled ELU）是 ELU 的缩放版本，具有"自归一化"特性——在特定条件下，SELU 网络的激活值会自动保持均值为 0、方差为 1，不需要 BatchNorm。推导这个结论需要 91 页数学附录（Slide 34），这体现了激活函数设计的理论深度。

GELU：Transformer 时代的新宠

GELU 的直觉（Slide 35）

GELU（Gaussian Error Linear Unit）的核心思想非常优雅：对于输入 x，以概率 P(X ≤ x) 保留它，以概率 1 - P(X ≤ x) 将它置零（其中 X ~ N(0,1)）。然后取这个随机操作的期望：

gelu(x) = x · P(X ≤ x) = x · Φ(x) ≈ x · σ(1.702x)

大的正值几乎必定被保留（P ≈ 1），大的负值几乎必定被丢弃（P ≈ 0）——这是一种数据依赖的 Dropout。GELU 在 BERT、GPT、ViT 等 Transformer 架构中被广泛使用。

Slide 37：激活函数的终极建议

场景推荐激活函数理由
默认选择ReLU简单、快速、够用
想要更好的性能Leaky ReLU / ELU / GELU可能多 0.1% 的提升
Transformer 架构GELUViT、BERT、GPT 的标配
绝对不要用Sigmoid / Tanh梯度消失问题太严重
实践策略 Justin 的原话（Slide 37）："Don't think too hard. Just use ReLU." 这是非常好的工程建议——激活函数的选择对最终性能的影响通常很小（Slide 36 显示不同激活函数在 CIFAR-10 上的差异不超过 1%）。把精力花在数据、架构和训练策略上，而不是纠结激活函数。
PART 04

数据预处理：为什么要 Zero-Center？

对应 Slides 38–44

数据预处理看似简单，但 Justin 在 Slide 40 特意回顾了 Sigmoid 的"非零中心"问题，说明预处理和激活函数的问题是同一个本质：

核心洞察（Slide 40） 如果输入数据 x 全是正数（比如图像像素值在 [0, 255]），那么第一层权重的梯度也会面临"全正或全负"的问题——和 Sigmoid 输出非零中心完全一样的数学原因。所以减去均值（zero-center）是必须的，它让输入数据有正有负，解放了梯度的符号。

图像数据的标准预处理流程

方法操作使用者
减去均值图像逐像素减去训练集均值（得到 [H,W,3] 均值图）AlexNet
减去通道均值逐通道减去均值（只需 3 个数字）VGGNet
标准化减去通道均值 & 除以通道标准差ResNet（最常用）

Slide 42 还提到了 PCA 和 Whitening（使数据协方差矩阵为单位阵），但明确指出图像领域很少使用，因为卷积网络本身就能学到类似的特征变换。

Slide 43 的几何解释 归一化前：loss 对权重矩阵的变化非常敏感，loss landscape 在不同方向上的曲率差异很大（椭圆形等高线），SGD 会走很多弯路。

归一化后：loss landscape 更"圆"（各方向曲率接近），SGD 可以更直接地走向最优解。这和 Lecture 4 中讲的"梯度下降在窄长峡谷中表现差"是同一个道理。 
# PyTorch 中的标准做法（以 ImageNet 为例）
import torchvision.transforms as T

transform = T.Compose([
    T.Resize(256),
    T.CenterCrop(224),
    T.ToTensor(),                              # 像素从 [0,255] 转为 [0,1]
    T.Normalize(mean=[0.485, 0.456, 0.406],    # ImageNet 训练集的通道均值
                std=[0.229, 0.224, 0.225])      # ImageNet 训练集的通道标准差
])
PART 05

权重初始化：从全零到 Xavier/Kaiming

对应 Slides 45–55

权重初始化是本讲的理论核心。Justin 通过一系列实验（Slides 50–55）展示了初始化的好坏直接决定模型能否训练。

初始化为零？

Slide 46–47：如果所有权重都初始化为 0，所有神经元的输出都相同（全是 0），梯度也全相同——所有神经元在每一步做完全一样的更新。这叫对称性无法打破（no symmetry breaking）。网络无法学习到有意义的表示。

小随机数？

Slide 48–49：用 std=0.01 的高斯分布初始化。对浅层网络可以工作，但深层网络会出问题。

Slides 50–55：激活值统计的可视化实验

Justin 构建了一个 6 层全连接网络（隐藏层 4096 维，Tanh 激活），观察前向传播时每层激活值的分布：

std=0.01 时：激活值逐层坍缩到零

权重太小 → 每层输出 = W × x，方差逐层缩小 → 到第 6 层激活值几乎全是 0。

后果：梯度 dL/dW 也趋近于 0（因为梯度公式中包含激活值），完全无法学习。

std=0.05 时：激活值饱和到 ±1

权重稍大 → 每层输出的绝对值越来越大 → Tanh 饱和在 ±1 → 局部梯度趋近于 0。

后果：和 Sigmoid 的梯度消失问题一样，同样无法学习。

这两个极端说明：初始化的方差既不能太小也不能太大。需要一个"刚刚好"的值——这就是 Xavier 初始化。

PART 06

Xavier 初始化的数学推导

对应 Slides 56–63

这是本讲中最重要的数学推导。理解它，你就能明白所有初始化方法的设计思路。

目标：让每层输出的方差等于输入的方差

考虑一个线性层 y = Wx（暂时忽略偏置和激活函数）。输出 y 的第 i 个元素：

yi = Σj=1Din xj wj

我们想要 Var(yi) = Var(xj)——输出的方差不放大也不缩小。

推导过程（Slides 59–63）

假设 x 和 w 独立同分布（iid），且都是零均值：

Step 1：Var(yi) = Var(Σ xjwj)

Step 2：= Din · Var(xjwj)  （因为各项独立）

Step 3：= Din · [E(xj2)E(wj2) - E(xj)2E(wj)2]  （x, w 独立）

Step 4：= Din · Var(xj) · Var(wj)  （x, w 零均值 ⇒ E(x)=0, E(w)=0）

Xavier 初始化推导的四步

要让 Var(yi) = Var(xj)，需要：

Din · Var(w) = 1  ⇒  Var(w) = 1/Din  ⇒  std(w) = 1/√Din
Xavier 初始化 (Glorot & Bengio, 2010)

权重从以下分布中采样：W ~ N(0, 1/Din)

或等价地：std = 1/√Din

对于卷积层：Din = kernel_size2 × in_channels

Step 3 到 Step 4 的关键跳跃

Var(XY) = E(X2)E(Y2) - [E(X)]2[E(Y)]2（当 X, Y 独立时）

当 E(X) = 0 时：E(X2) = Var(X) + [E(X)]2 = Var(X)

所以 Var(XY) = Var(X) · E(Y2) = Var(X) · Var(Y) （当两者均零均值时）

这个推导依赖于"零均值"假设——而 Tanh 恰好是零均值的激活函数。这就是为什么 Xavier 对 Tanh 效果好。

Slide 57 的实验验证：使用 Xavier 初始化后，6 层网络每层的激活值分布几乎一样——"Just right"。方差既没有爆炸也没有消失。

PART 07

Kaiming 初始化与 ResNet 的特殊处理

对应 Slides 64–69

Xavier 对 ReLU 失效的原因

Slide 64–65 展示了一个关键的实验：把激活函数从 Tanh 换成 ReLU，使用 Xavier 初始化——激活值又坍缩到零了！

为什么 Xavier + ReLU 不工作？ Xavier 的推导假设激活函数是零中心的（E[σ(x)] = 0）。但 ReLU 的输出永远 ≥ 0，不满足零均值假设。具体来说，ReLU 会将大约一半的激活值变为 0，相当于每层输出的方差被砍了一半。经过多层后，方差指数级缩小到接近零。

Kaiming 初始化（He et al., 2015）

修复方法很简单：ReLU 每层砍一半方差，那就把初始方差乘以 2 来补偿：

std = √(2/Din)

这就是 Kaiming 初始化（也叫 MSRA 初始化或 He 初始化）。Slide 66 显示使用 Kaiming 初始化后，6 层 ReLU 网络的激活值分布恢复正常。

# PyTorch 中的用法
import torch.nn as nn

# 方法一：手动初始化
nn.init.kaiming_normal_(conv.weight, mode='fan_in', nonlinearity='relu')

# 方法二：PyTorch 的 nn.Linear 和 nn.Conv2d 默认就使用 Kaiming 初始化
# 所以大多数时候你不需要手动设置

ResNet 的初始化技巧（Slides 67–68）

Residual Block 的方差增长问题

ResNet 的残差块计算 output = F(x) + x。如果 F(x) 和 x 的方差都是 Var(x)，那么输出的方差约为 2 · Var(x)（假设 F(x) 和 x 不相关）。

经过多个残差块，方差会逐块翻倍，最终爆炸。

解决方案：将每个残差块中第二个卷积层的权重初始化为零。这样初始时 F(x) = 0，output = 0 + x = x，方差完全不变。随着训练进行，网络会自然地学习非零的 F(x)。

这个技巧后来被 Fixup（Zhang et al., 2019）等工作进一步发展，甚至可以训练没有 BatchNorm 的深层 ResNet。

初始化方法公式适用激活函数
Xavierstd = 1/√DinTanh, Sigmoid（零中心或对称的）
Kaimingstd = √(2/Din)ReLU（有一半被截断的）
ResNet 第二层 convW = 0残差块中保持方差稳定
PART 08

正则化：L2、Dropout 与通用模式

对应 Slides 70–87

模型能训练了，但开始过拟合（training loss 低，validation loss 高）。正则化就是解决这个问题的工具。

经典正则化：L2 / L1 / Elastic Net

在损失函数中加入惩罚项（Slide 71）：

Ltotal = Ldata + λ · R(W)
  • L2 正则化（Weight Decay）：R(W) = Σ W2，鼓励权重趋小，防止任何一个权重过大
  • L1 正则化：R(W) = Σ |W|，鼓励稀疏性，让部分权重变为精确的 0
  • Elastic Net：L1 + L2 的组合

Dropout（Slides 72–82）

Dropout 是深度学习中最经典的正则化技术之一。核心思想极其简单：

Dropout 的机制 训练时：每次前向传播，以概率 p（通常 p=0.5）随机将每个神经元的输出置为 0。
测试时：不丢弃任何神经元，但将所有输出乘以 (1-p) 来补偿。

Dropout 为什么有效？两种解释

解释一（Slide 74）：防止特征共适应

如果网络可以依赖于特定的神经元组合来做预测（比如"有耳朵 + 有尾巴 + 有毛 = 猫"），Dropout 强迫网络不能依赖任何单一特征——因为每个特征都可能被随机丢弃。网络必须学习冗余的、分布式的表示。

解释二（Slide 75）：隐式模型集成

每个 dropout mask（哪些神经元被丢弃）定义了一个不同的"子网络"。一个有 4096 个神经元的 FC 层，可能的 mask 有 24096 ≈ 101233 种——比宇宙中的原子还多（约 1082）！Dropout 相当于在训练一个天文数字大小的模型集成。

Dropout 的测试时缩放（Slides 76–82）

Slide 77–79 用一个单神经元的例子精彩地推导了测试时为什么要缩放：

考虑神经元 a = w1x + w2y，dropout 概率 p=0.5：

测试时（无 dropout）：E[a] = w1x + w2y

训练时（有 dropout）：E[a] = ¼(w1x + w2y) + ¼(w1x + 0) + ¼(0 + 0) + ¼(0 + w2y)

         = ½(w1x + w2y)

训练时的期望输出是测试时输出的 p 倍

训练时的期望输出 = p × 测试时输出。为了让测试时行为和训练时一致，测试时需要将输出乘以 p（或等价地，训练时将存活的神经元除以 p，这就是Inverted Dropout）。

Inverted Dropout（Slide 82）——实践中的标准做法

实践中几乎总是用 Inverted Dropout：训练时丢弃后立即除以 (1-p)，测试时完全不做任何操作。这样测试时代码更简洁，性能也更好。

# 训练时
mask = (torch.rand_like(h) > p).float()  # 随机 mask
h = h * mask / (1 - p)                   # 丢弃 + 缩放

# 测试时
# 什么都不做！直接用 h

Dropout 在现代架构中的地位（Slides 83–84）

AlexNet 和 VGG 的大部分参数在全连接层中——Dropout 主要应用在那里。但现代架构（GoogLeNet、ResNet 等）使用 Global Average Pooling 替代了大型 FC 层，参数量大幅减少，通常不需要 Dropout。

正则化的通用模式（Slides 85–87）

统一框架

几乎所有正则化技术都遵循同一模式：

训练时：加入某种随机性 → y = f(x, z) （z 是随机变量）

测试时：对随机性取平均（有时是近似的）→ y = Ez[f(x, z)]

Dropout 的 z 是随机 mask；BatchNorm 的 z 是随机 mini-batch 的统计量；数据增强的 z 是随机变换。统一框架让你能更快地理解新的正则化技术。

Slide 87 的关键结论：对于 ResNet 及之后的架构，L2 + BatchNorm 通常就够了。

PART 09

数据增强：最实用的正则化手段

对应 Slides 88–96

数据增强是最便宜、最有效的正则化方法——不增加模型复杂度，不增加训练时间（可以忽略），却能显著提升泛化能力。

核心思想

在训练时，对每张输入图像应用随机变换（翻转、裁剪、颜色抖动等），生成"新"的训练样本。关键约束：变换不能改变图像的语义（一张猫的水平翻转仍然是猫）。

标准增强技术

技术操作Slides
水平翻转以 50% 概率水平翻转图像90
随机裁剪 + 缩放随机缩放到不同尺寸，再裁剪固定大小的 patch91–92
颜色抖动随机改变亮度、对比度、饱和度93
RandAugment随机组合多种变换（旋转、剪切、色彩变换等）94–95

Slide 91–92 详细描述了 ResNet 的数据增强策略：

训练时

  1. 随机选择 L ∈ [256, 480]
  2. 将短边 resize 到 L
  3. 随机裁剪 224×224 的 patch
  4. 随机水平翻转

测试时

  1. 将图像 resize 到 5 个尺度：{224, 256, 384, 480, 640}
  2. 每个尺度裁剪 10 个 224×224 的 patch（4 角 + 中心 + 翻转）
  3. 对 50 个 patch 的预测取平均
Slide 96：为你的任务设计增强策略

数据增强编码了你对任务的先验知识。问自己："输入的什么变化不应该改变输出？"

图像分类：水平翻转可以，但垂直翻转通常不行（现实世界的照片很少是上下颠倒的）

医学图像：旋转和翻转通常都可以（细胞没有"上下"之分）

文字识别：轻微旋转可以，但水平翻转会改变文字含义

不同任务需要不同的增强策略——这里没有万能方案。

PART 10

高级正则化技术全景

对应 Slides 97–107

Justin 在 Slides 97–106 快速介绍了一系列高级正则化技术。它们都遵循同一个模式：训练时加随机性，测试时取平均。

技术清单

技术随机性来源核心思想
DropConnect随机将权重置零（而非激活值）Dropout 的权重版本
Fractional Max Pooling随机化池化区域的大小和位置池化操作本身也可以加入随机性
Stochastic Depth随机跳过 ResNet 中的某些残差块训练时用"浅"网络，测试时用完整网络
Cutout / Random Erasing随机遮挡图像的一个矩形区域强迫网络不依赖局部特征
Mixup随机混合两张图像的像素和标签线性插值：40%猫+60%狗
CutMix将一张图像的矩形区域替换为另一张图像Cutout + Mixup 的结合
Label Smoothing将 one-hot 标签"软化"[1,0,0] → [0.9, 0.05, 0.05]
Stochastic Depth（Slide 100–101）

Stochastic Depth 的想法和 Dropout 类似，但它丢弃的是整个残差块而非单个神经元。训练时每个残差块以一定概率被跳过（直接 output = x），测试时使用完整网络。

这个技术近年来越来越流行，被用在 EfficientNet、DeiT 等现代架构中。它不仅有正则化效果，还能加速训练（被跳过的块不需要计算前向和反向传播）。

Mixup 与 CutMix（Slides 103–105）

Mixup：随机采样混合比例 λ ~ Beta(α, α)，然后：

x̃ = λ · x1 + (1-λ) · x2  ỹ = λ · y1 + (1-λ) · y2

CutMix：不是像素级混合，而是将 x1 的一个矩形区域替换为 x2 的对应区域。标签按面积比例混合。CutMix 通常比 Mixup 效果更好，因为混合后的图像更"自然"。

Label Smoothing（Slide 106）

标准的 one-hot 标签要求模型对正确类别输出 100% 的概率——这可能导致过度自信。Label Smoothing 将目标概率从 [1, 0, 0, ...] 变成 [1-ε+ε/K, ε/K, ε/K, ...]（K 是类别数，ε 通常为 0.1）。

效果：模型的输出概率更校准（calibrated），泛化能力通常更好。

Slide 107：正则化总结与推荐

实践建议
  • 大型 FC 层：使用 Dropout
  • 数据增强：永远是好主意，几乎没有代价
  • CNN（ResNet 等）：L2 + BatchNorm 通常足够
  • 想要更好的性能？尝试 Cutout、MixUp、CutMix、Stochastic Depth、Label Smoothing
  • Transformer（ViT 等）：不用 BatchNorm，改用 LayerNorm + 其他正则化

COMPETITIVE EDGE

在 Vibe Coding 时代，为什么这些知识仍然重要？

在 AI 可以自动生成训练代码的时代，以下场景是 AI 无法替你处理的：

场景一：模型不收敛

你用 AI 生成了一个训练脚本，loss 从一开始就不下降。AI 可能会建议"降低学习率"——但真正的问题可能是权重初始化不当（Part 5–7）。你需要检查每层激活值的统计量，判断是坍缩到零（初始化太小）还是饱和（太大），然后选择正确的初始化方案。

场景二：训练集 98%，验证集 70%

明显的过拟合。AI 可能会建议"加 Dropout"——但如果你用的是 ResNet，Dropout 反而可能降低性能（Part 8）。正确的策略可能是加强数据增强（Part 9）或使用 Label Smoothing（Part 10）。理解每种正则化技术的适用场景，比盲目套用模板重要得多。

场景三：梯度变成 NaN

这通常是激活值爆炸导致的。你需要理解方差在网络中的传播（Part 6 的数学推导），才能诊断出问题出在哪一层、为什么爆炸、如何修复。这是纯粹的数学能力，AI 代码生成帮不了你。
