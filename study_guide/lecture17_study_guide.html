PART 01

全局定位：从 RNN 到 Attention

对应 Slides 1–4

Lecture 16 建立了 RNN（循环神经网络）的完整框架——你学会了如何用隐藏状态 ht 来处理变长序列，用 LSTM 解决梯度消失问题。但 RNN 有一个深层的架构缺陷：它必须把整个输入序列的信息压缩进一个固定大小的向量。Lecture 17 回答的是："如何打破这个信息瓶颈？"——答案就是 Attention。

PART 02

Seq2Seq 的瓶颈问题

对应 Slides 5–11

回顾 Seq2Seq（序列到序列）架构：Encoder（编码器）把输入序列 x1, ..., xT 逐步编码到隐藏状态 h1, ..., hT，最终把所有信息压缩到最后一个隐藏状态 hT；Decoder（解码器）从 hT 出发，逐步生成输出序列。

标准 Seq2Seq：整个输入句子的语义被压缩成一个固定长度的向量 c——这就是信息瓶颈
核心问题：信息瓶颈

Context vector c（通常就是 hT）是一个固定大小的向量（比如 256 维或 512 维）。无论输入是 5 个单词还是 1000 个单词，都必须压缩进同一个维度。当 T 很大时，早期输入的信息在经过多步 RNN 后几乎完全丢失——这就是 Sutskever et al. (2014) 的 Seq2Seq 论文在长句子上性能急剧下降的原因。

Justin 在 Slide 11 提出了一个自然的想法：为什么非要用一个 context vector？为什么不能在每个解码步都"回头看"编码器的所有隐藏状态？——这就是 Attention 的动机。

PART 03

Attention 机制：让解码器"回头看"

对应 Slides 12–25 (Bahdanau et al., ICLR 2015)

Attention 的核心想法惊人地简洁：在解码器的每一步 t，不再使用一个固定的 context vector c，而是动态计算一个新的 context vector ct，它是所有编码器隐藏状态的加权和。权重由"当前解码器状态 st-1 和每个编码器状态 hi 之间的相关度"决定。

三步走：Attention 的计算流程（Slide 13–15）

①
计算对齐分数（Alignment Scores） et,i = fatt(st-1, hi) — 用一个小 MLP 计算解码器当前状态 st-1 和每个编码器隐藏状态 hi 之间的"相关度"。et,i 是一个标量。
②
归一化为注意力权重（Attention Weights） at,i = softmax(et,: )i — 将所有对齐分数过 softmax，得到 0 到 1 之间的权重，且 ∑i at,i = 1。
③
计算 Context Vector ct = ∑i at,i hi — 编码器隐藏状态的加权平均。权重大的 hi 贡献多，权重小的贡献少。
Attention 完整公式
et,i = fatt(st-1, hi)
at = softmax(et)
ct = ∑i at,i · hi
st = gU(yt-1, st-1, ct)
ct 在每个时间步 t 都不同——解码器在不同时刻"看"输入序列的不同部分

直觉：翻译中的注意力对齐（Slides 21–24）

在英法翻译 "The agreement on the European Economic Area was signed in August 1992" 中，attention 权重的可视化（热力图）揭示了三个有趣的现象：

对角线模式

词序相同时，attention 权重沿对角线分布——说明模型学会了"逐词对齐"

词序调换

当两种语言词序不同（如英语 "European Economic Area" → 法语 "zone économique européenne"），注意力自动调整

多对一映射

动词变位（如 "was signed" → "a été signé"）需要同时关注多个源词，attention 也能处理

为什么 Attention 权重不需要监督 Slide 15 特别强调：整个 attention 过程是完全可微分的（differentiable）。我们不需要告诉模型"翻译 estamos 时应该看 we 和 are"——通过端到端的反向传播，模型自己学会了在哪里"看"。这是一个非常优雅的设计：attention 权重是隐式学习的，不需要额外标注。
关键洞察：解码器其实把输入当作"集合"而非"序列" Slide 25 指出，解码器在计算 attention 时，并不在意 hi 的顺序——它对 h1, ..., hT 做加权求和，无论它们的排列如何，结果都一样。换句话说，attention 操作本质上是在一个无序集合 {hi} 上进行的。这个观察在后面引出 self-attention 时至关重要。
PART 04

图像描述中的 Attention

对应 Slides 26–43 (Xu et al., ICML 2015)

Attention 不局限于文本到文本的翻译——只要有一组"输入向量"和一个"查询"，attention 就能工作。在图像描述（Image Captioning）中，"输入向量"变成了 CNN 特征图的空间位置，"查询"变成了 RNN 解码器的隐藏状态。

图像描述：CNN 特征图的每个空间位置就像一个"单词"，attention 机制选择当前要关注的图像区域
类比人类视觉系统：中央凹与眼跳 Justin 在 Slides 39–42 引入了一个精彩的类比。人眼的视网膜上有一个叫中央凹（fovea）的微小区域，只有这里能看到高清细节。为了"看清"整个场景，眼球会不断快速跳动（saccade），每次把中央凹对准不同的位置。Attention 机制做的事情几乎一模一样：每个解码步就像一次"眼跳"，把计算资源集中在最相关的输入位置上。

Justin 随后列出了一系列 "X, Attend, and Y" 的论文（Slide 43），证明 Attention 已经成为一种通用的架构模式——不只是翻译或图像描述，而是几乎所有"从一个东西生成另一个东西"的任务都可以用。

PART 05

通用 Attention Layer 的演进

对应 Slides 44–55

既然 Attention 这么通用，Justin 开始把它抽象成一个独立的神经网络层。这个抽象过程是 Lecture 17 最精华的部分——它展示了如何从一个具体的应用场景出发，逐步提炼出一个通用的计算原语。

抽象的起点：Attention Layer v1（Slide 44）

Attention Layer — 基础版 BASE

输入：Query 向量 q (DQ)，Input 向量集合 X (NX × DX)，相似度函数 fatt

计算：
相似度：ei = fatt(q, Xi)  (NX 个标量)
权重：a = softmax(e)  (NX 个概率)
输出：y = ∑i ai Xi  (DX 维向量)

从 Slide 44 到 Slide 55，Justin 对这个基础版做了四次改进。每次改进都有明确的动机：

改进一：用点积代替 MLP（Slide 45）

原版的 fatt 是一个 MLP，但点积 ei = q · Xi 更简洁、更高效——矩阵乘法，GPU 友好。

代价：要求 q 和 Xi 的维度必须相同（DQ = DX）。实际中用 Key/Value 分离来解决。

改进二：缩放点积（Slide 46–47）

Scaled Dot Product
ei = (q · Xi) / √DQ

为什么要除以 √D？如果 q 和 Xi 的每个元素都是常数 a，那么点积 = D · a²，随维度线性增长。高维下点积值会很大，导致 softmax 饱和（梯度消失）。除以 √D 把方差拉回 1。

面试高频考点 "为什么 Transformer 里要 scale by √dk？" 是 ML 面试经典题。答案要点：防止 softmax 在高维空间中因为输入过大而饱和，导致梯度消失。

改进三：多个 Query（Slide 48）

之前只有一个 query 向量 q。但实际中我们往往想同时查询多个位置（比如一句话的所有词都想看一下输入）。把 q 扩展为矩阵 Q (NQ × DQ)，计算变成矩阵乘法：

E = Q XT / √DQ   A = softmax(E, dim=1)   Y = AX

现在一次前向传播就能并行计算 NQ 个 attention 输出——GPU 利用率大幅提升。

改进四：Key-Value 分离（Slide 49–55）

核心思想："用什么来匹配"和"匹配到了取什么"可以是不同的东西。引入两个可学习的投影矩阵：

  • Key 矩阵 WK：把 X 投影到"索引空间"，用于和 Query 做匹配
  • Value 矩阵 WV：把 X 投影到"内容空间"，被匹配到后用于输出
最终版：Scaled Dot-Product Attention with Key-Value
K = X WK,   V = X WV
E = Q KT / √DQ
A = softmax(E, dim=1)
Y = A V
数据库的类比 想象一个键值数据库：你拿着一个 Query（搜索词），和数据库中每个条目的 Key（索引）比较相似度，然后取出匹配条目的 Value（内容）。Attention 就是这个过程的"软"版本——不是精确匹配一个 key，而是对所有 key 做加权平均。
PART 06

Self-Attention：序列自身的交互

对应 Slides 56–70

前面的 Attention 都是"一个序列查询另一个序列"——Query 来自解码器，Key/Value 来自编码器。Self-Attention 的突破性想法是：让序列和它自己做 attention。

Cross-Attention

Q 来自解码器 ≠ X（Key/Value）来自编码器
两个不同的序列之间交互

→
Self-Attention

Q, K, V 全部来自同一个输入 X
Q = XWQ, K = XWK, V = XWV

Self-Attention Layer SELF

输入：X (N × DX)，WQ (DX × DQ)，WK (DX × DQ)，WV (DX × DV)

Q = X WQ  (N × DQ)
K = X WK  (N × DQ)
V = X WV  (N × DV)
E = Q KT / √DQ  (N × N)
A = softmax(E, dim=1)  (N × N)
Y = A V  (N × DV)

注意 E 的形状是 N×N——每个向量和每个向量之间都有交互！这是 self-attention 的力量，也是它内存开销的来源。

核心工程细节：张量形状追踪 这是你在作业和面试中必须掌握的——能准确追踪每一步的张量形状：
变量形状含义
XN × DXN 个输入向量
QN × DQN 个 query 向量
KN × DQN 个 key 向量
VN × DVN 个 value 向量
E = QKTN × N所有 Q-K 对的相似度
AN × Nattention 权重矩阵
Y = AVN × DVN 个输出向量

Self-Attention 的代码实现

# PyTorch 风格伪代码
def self_attention(X, W_Q, W_K, W_V):
    # X: (N, D_X)
    Q = X @ W_Q          # (N, D_Q)
    K = X @ W_K          # (N, D_Q)
    V = X @ W_V          # (N, D_V)

    D_Q = Q.shape[-1]
    E = Q @ K.T / (D_Q ** 0.5)   # (N, N)
    A = torch.softmax(E, dim=-1) # (N, N)
    Y = A @ V                     # (N, D_V)
    return Y
Self-Attention 的核心性质：置换等变性（Permutation Equivariance）

Justin 用 Slides 63–69 详细证明了一个重要性质：如果我们打乱输入向量的顺序，输出向量也会以完全相同的方式被打乱。数学上：f(s(X)) = s(f(X))，其中 s 是任意置换。

这意味着 self-attention 天然不知道向量的顺序——它把输入当作一个无序集合来处理。这是 RNN 和 self-attention 的根本区别：RNN 天然有序，self-attention 天然无序。

PART 07

位置编码与 Masked Attention

对应 Slides 71–73

既然 self-attention 不知道顺序，但语言和图像显然是有顺序的，怎么办？答案是显式注入位置信息。

位置编码（Positional Encoding）
X'i = Xi + E(i)
E(i) 可以是可学习的查找表（learned lookup table），也可以是固定的三角函数（sinusoidal）

Learned Positional Encoding

为每个位置学一个 D 维向量。简单有效，但只能处理训练时见过的位置数（如最多 512）。

Sinusoidal Positional Encoding

用不同频率的正弦/余弦函数。理论上可外推到任意长度，但实际效果不总是更好。

Masked Self-Attention（Slide 72–73）

在语言建模（预测下一个词）中，模型不应该看到未来的信息。Masked Self-Attention 通过在 E 矩阵的上三角部分填充 −∞ 来实现——经过 softmax 后，这些位置的权重变成 0。

# Masked Self-Attention 的关键代码
E = Q @ K.T / (D_Q ** 0.5)       # (N, N)
mask = torch.triu(torch.ones(N, N), diagonal=1).bool()
E[mask] = float('-inf')        # 上三角设为 -inf
A = torch.softmax(E, dim=-1)     # 上三角变成 0
为什么用 -inf 而不是 0 如果直接把 E 的某些位置设为 0，经过 softmax 后这些位置的权重不是 0（因为 e0 = 1 > 0）。必须设为 −∞，因为 e−∞ = 0，这样 softmax 后这些位置的权重才是真正的 0。这个细节在代码实现中很容易出错。
PART 08

Multi-Head Attention

对应 Slides 74–87

一个 self-attention head 只能学习一种关注模式。比如在理解一个句子时，你可能需要同时关注"语法关系"、"语义相关性"、"指代关系"等多种不同的关系。Multi-Head Attention（MHA）用多个 head 并行计算，每个 head 学习不同的注意力模式。

Multi-Head Self-Attention H HEADS

对每个 head h = 1, ..., H：
 Q(h) = X WQ(h),   K(h) = X WK(h),   V(h) = X WV(h)
 Y(h) = Attention(Q(h), K(h), V(h))  (N × DV/H)

合并所有 head：
 Y = Concat(Y(1), ..., Y(H)) WO  (N × Dout)

实现技巧：Split 而非 Copy（Slide 75–80）

一个关键的工程细节：我们不是把每个 head 都分配 D 维的 Q/K/V（那样参数量会乘以 H），而是把 D 维均分给 H 个 head，每个 head 只操作 D/H 维。

# Multi-Head Attention 的高效实现
class MultiHeadAttention(nn.Module):
    def __init__(self, D, H):
        self.H = H
        self.D_head = D // H
        self.W_Q = nn.Linear(D, D)   # 一次性投影所有 head
        self.W_K = nn.Linear(D, D)
        self.W_V = nn.Linear(D, D)
        self.W_O = nn.Linear(D, D)

    def forward(self, X):
        N, D = X.shape
        # 投影后 reshape 成多个 head
        Q = self.W_Q(X).view(N, self.H, self.D_head)  # (N, H, D/H)
        K = self.W_K(X).view(N, self.H, self.D_head)
        V = self.W_V(X).view(N, self.H, self.D_head)
        # 每个 head 独立做 attention（batch matmul）
        E = torch.einsum('nhd,mhd->hnm', Q, K) / (self.D_head**0.5)
        A = torch.softmax(E, dim=-1)          # (H, N, N)
        Y = torch.einsum('hnm,mhd->nhd', A, V)
        # 合并所有 head
        Y = Y.reshape(N, D)
        return self.W_O(Y)
为什么需要多头？

单个 attention head 的 softmax 输出是一个概率分布——它只能表达一种"聚焦模式"。但语言中的关系是多维的：主语-动词、形容词-名词、代词-先行词……多个 head 让模型同时捕捉多种不同的关系。实际中，研究者发现不同的 head 确实学到了不同的语言学特征。

PART 09

三种序列处理范式对比

对应 Slides 88–91

Justin 在 Slides 88–91 做了一个非常重要的对比，把三种处理序列的方式放在一起。这个对比是理解"为什么 Transformer 赢了"的关键。

RNN

操作对象：有序序列

✓ 长程依赖：hT 看过全部输入

✗ 不可并行：必须顺序计算 ht

隐藏状态 ht 是序列压缩的"记忆"，信息会衰减

1D 卷积

操作对象：多维网格

✗ 长程依赖：需要堆叠很多层

✓ 高度并行：所有输出可同时算

感受野受限于卷积核大小 × 层数

Self-Attention

操作对象：向量集合

✓ 长程依赖：一层就看到全部

✓ 高度并行：所有输出可同时算

代价：内存 O(N²)，N 是序列长度

Self-Attention 的 O(N²) 问题

注意力矩阵 E 的形状是 N × N。如果序列长度 N = 4096（GPT-3 的上下文长度），那么一个 attention 层就需要存储 4096² ≈ 1600 万个数——乘以 batch size、head 数量和层数后，内存开销巨大。这就是为什么后续研究出现了大量的"高效 Attention"变体（如 Flash Attention、Linear Attention 等）。

Transformer 架构详解

对应 Slides 92–102 (Vaswani et al., NeurIPS 2017)

终于到了这门课中最重要的单个架构——Transformer。"Attention Is All You Need" 这篇论文的核心洞察是：完全抛弃 RNN 和 CNN，仅用 attention 构建整个模型。

Transformer Block 的结构（Slides 92–99）

一个 Transformer Block 由以下组件从下到上堆叠而成：

一个 Transformer Block 的完整结构。注意只有 Self-Attention 层涉及向量间交互，其余操作都是逐向量独立的。
Transformer Block 的设计哲学（Slide 99）

Transformer Block 的设计非常精妙。Self-Attention 是唯一的"交互操作"——它让所有向量之间互相通信。Layer Norm 和 MLP 都是逐向量独立的——它们只处理每个向量自己的内部表示。加上 Residual Connection 保证了梯度流通。这种"全局交互 + 局部变换"的交替设计使得 Transformer 极度可并行化、可扩展。

Pre-Norm vs Post-Norm（Slides 100–101）

Post-Norm（原始论文）

LayerNorm 在 residual 之后：
y = LN(x + SubLayer(x))

原始 Transformer 用这种设计。训练可能不稳定，通常需要 warmup。

Pre-Norm（现代主流）

LayerNorm 在 SubLayer 之前：
y = x + SubLayer(LN(x))

梯度直接流过 residual path，训练更稳定，常用于 GPT 等大模型。

工程重要性 Pre-Norm vs Post-Norm 看起来只是移动了一层 Layer Norm 的位置，但对训练稳定性影响巨大。在实际大模型训练中，Pre-Norm 是默认选择（GPT-2/3, LLaMA 等都用 Pre-Norm）。面试中被问到"Transformer 训练不稳定怎么办"，Pre-Norm 是关键答案之一。

Justin 提到了一个重要的历史类比：就像 2012 年 AlexNet 在 ImageNet 上的突破证明了"大规模预训练 + 微调"在 CV 中的威力一样，BERT（2018）做了同样的事情，证明了这个范式在 NLP 中同样适用。Transformer 之所以能统治深度学习，不仅因为它的架构优雅，更因为它天然适合 scale——高度可并行、没有顺序瓶颈、增加参数就能增加表达能力。

2022 → 2026 的后续发展

Justin 的课停在了 GPT-3 和 Gopher。在那之后，这个 scaling 曲线还在继续：GPT-4、Claude、Gemini、LLaMA 等模型把参数量、训练数据和计算量推向了更极端的水平。Lecture 17 让你理解了这些系统的基础架构——所有这些大模型的核心都是这里学到的 Transformer Block（带一些改进如 RoPE、GQA、SwiGLU 等）。

COMPETITIVE EDGE

Vibe Coding 时代的竞争力

场景：AI 生成的 Transformer 代码有 bug，attention 输出全是 NaN AI 经常忘记 / math.sqrt(d_k) 这个 scaling 步骤，或者忘记在 masked attention 中用 -inf。当你看到 softmax 输出全是 NaN 或者全是 1/N 的均匀分布时，你需要知道 attention 机制的每一步在做什么，才能定位问题——是 scale 忘了？mask 方向反了？还是 softmax 的 dim 参数搞错了？这种调试能力不是 AI 能替代的。
场景：面试中被问"解释一下 Multi-Head Attention" 浅层回答是"多个 head 并行做 attention 然后 concat"。深层回答是：能解释 Q/K/V 投影的数学含义、scale 的方差分析、为什么 split 维度而不是复制全维度、以及 N×N 注意力矩阵的内存瓶颈和 Flash Attention 等解决方案。能从数学公式到 PyTorch 代码到系统优化全栈打通，才能在面试中脱颖而出。
场景：需要设计一个自定义 attention 变体 当你理解了 Attention 从"给 RNN 加的一个 hack"到"通用计算原语"的抽象过程后，你就有能力设计自己的 attention 变体——比如窗口 attention（Swin Transformer）、线性 attention（降低到 O(N) 复杂度）、交叉 attention（多模态融合）。理解底层原理才能创新，而不是只会调用 nn.MultiheadAttention。
